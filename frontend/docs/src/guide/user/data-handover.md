# 数据交接

数据交接模块用于管理用户离职交接和业务数据移交，确保业务连续性和数据完整性。

## 谁能使用数据交接？

| 功能     | 管理员 | 主任 | 团队负责人 | 财务 | 律师 | 行政 | 实习 |
| -------- | :----: | :--: | :--------: | :--: | :--: | :--: | :--: |
| 数据交接 |   ✅   |  ✅  |     ✅     |  ❌  |  ❌  |  ❌  |  ❌  |

::: tip 说明数据交接功能主要由管理层使用，用于处理人员变动时的业务数据移交。:::

---

## 进入数据交接

点击左侧菜单「数据交接」

---

## 交接类型说明

| 类型     | 说明                     | 适用场景       |
| -------- | ------------------------ | -------------- |
| 离职交接 | 员工离职时的全面数据移交 | 员工离职、退休 |
| 项目移交 | 特定项目的移交           | 项目负责人变更 |
| 客户移交 | 客户关系的移交           | 客户负责人变更 |
| 案源移交 | 案源线索的移交           | 案源跟进人变更 |

## 场景一：创建数据交接

当需要进行数据移交时：

1. 点击「新建交接」按钮
2. 系统打开3步骤向导：

**步骤1 - 确认交接信息**

| 字段 | 说明 | 必填 |
| --- | --- | :-: |
| 移交人 | 自动显示当前登录用户 | ✅ |
| 交接类型 | 离职交接（全部移交）/项目移交（指定项目）/客户移交（指定客户） | ✅ |
| 交接原因 | 说明交接原因 | ❌ |

3. 点击「下一步：预览数据」

**步骤2 - 预览数据**

系统自动统计当前用户名下的数据：

- 主办项目数量
- 负责客户数量
- 跟进案源数量
- 待办任务数量

页面显示详细列表：

- 主办项目列表（项目名称、编号、状态）
- 负责客户列表（客户名称、编号）

4. 确认数据无误后，点击「下一步：选择接收人」

**步骤3 - 选择接收人**

| 字段 | 说明 | 必填 |
| --- | --- | :-: |
| 接收人 | 选择接收数据的用户（按部门筛选） | ✅ |
| 包含案源人身份 | 勾选后案源人身份也会移交（涉及提成分配，仅离职交接显示） | ❌ |
| 备注 | 备注信息 | ❌ |

5. 点击「提交交接单」

::: warning 审批流程提交后将创建交接单，需要管理员审批后才会执行数据迁移。:::

---

## 场景二：查看交接详情

1. 在交接列表中找到交接单
2. 点击「详情」查看完整信息
3. 详情页显示：
   - 交接单号、交接类型、状态
   - 移交人、接收人、提交人
   - 交接原因
   - 提交时间、确认人、确认时间
   - 移交的数据明细（项目数、客户数、案源数、任务数）
   - 详细变更记录表格（数据类型、编号、名称、变更字段、原值、新值、状态）

---

## 场景三：审批数据交接

数据交接需要审批后才能执行：

1. 在审批中心找到待审批的交接申请
2. 查看交接详情
3. 选择审批结果：
   - **通过**：同意交接，状态变为「审批通过待执行」
   - **拒绝**：拒绝交接，需要填写拒绝原因

---

## 场景四：确认执行交接

审批通过后，需要确认执行：

1. 找到状态为「审批通过待执行」的交接单
2. 点击「确认执行」
3. 在确认弹窗中点击「确认执行」
4. 系统会：
   - 将项目的负责人/参与人变更为接收人
   - 将客户的负责人变更为接收人
   - 将案源的跟进人变更为接收人
   - 将任务的负责人变更为接收人
5. 交接完成，状态变为「已确认」

::: warning 执行不可逆确认执行后，数据迁移会立即生效，请确认无误后再执行。:::

---

## 场景五：取消交接

如果不需要执行交接了：

1. 找到状态为「待审批」或「审批通过待执行」的交接单
2. 点击「取消」
3. 确认后交接单状态变为「已取消」

::: tip 取消限制只有「待审批」和「审批通过待执行」状态的交接单可以取消。已确认的交接不能取消。:::

---

## 交接状态说明

| 状态           | 含义                   | 颜色 | 可执行操作     |
| -------------- | ---------------------- | ---- | -------------- |
| 待审批         | 等待审批人处理         | 蓝色 | 取消           |
| 审批通过待执行 | 审批通过，等待确认执行 | 橙色 | 确认执行、取消 |
| 已拒绝         | 审批被拒绝             | 红色 | -              |
| 已确认         | 交接已完成             | 绿色 | -              |
| 已取消         | 交接已取消             | 灰色 | -              |

## 常见问题

**Q: 离职交接和项目移交有什么区别？** A: 离职交接是全面的数据移交，包括该员工名下的所有项目、客户、案源和任务。项目移交只针对特定项目。

**Q: 交接后原负责人还能看到数据吗？** A: 交接完成后，原负责人将不再是数据的负责人，但如果他仍是项目的参与人，可能仍有查看权限。

**Q: 可以把数据交接给多个人吗？** A: 一次交接只能指定一个接收人。如果需要分配给多人，需要创建多个交接单。

**Q: 交接后发现有遗漏怎么办？** A: 可以再创建一个新的交接单，补充移交遗漏的数据。

**Q: 交接单号是怎么生成的？** A: 系统自动生成，格式为 `JJ + 年月日 + 序号`。

**Q: 移交人和接收人可以是同一个人吗？** A: 不可以，系统会校验移交人和接收人不能相同。
