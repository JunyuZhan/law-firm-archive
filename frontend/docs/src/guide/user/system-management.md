# 系统管理

系统管理模块用于管理系统的基础配置，包括用户、角色、部门、菜单、系统配置和操作日志等。

## 谁能使用系统管理？

| 功能 | 管理员 | 主任 | 团队负责人 | 财务 | 律师 | 行政 | 实习 |
|------|:-----:|:----:|:--------:|:----:|:----:|:----:|:----:|
| 用户管理 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 角色管理 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 部门管理 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 菜单管理 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 系统配置 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 操作日志 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |

::: warning 管理员专属
系统管理功能仅限管理员使用，其他角色无法访问。
:::

## 用户管理

### 进入用户管理

点击左侧菜单「系统管理」→「用户管理」

---

### 场景一：创建新用户

1. 点击「新增用户」按钮
2. 填写用户信息：

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 用户名 | 登录账号，创建后不可修改 | ✅ |
| 姓名 | 用户真实姓名 | ✅ |
| 密码 | 初始密码 | ✅ |
| 部门 | 所属部门 | ✅ |
| 职位 | 职位名称 | ❌ |
| 手机号 | 联系电话 | ❌ |
| 邮箱 | 电子邮箱 | ❌ |
| 角色 | 分配角色（可多选） | ✅ |

3. 点击「确定」保存

::: tip 初始密码
建议使用统一的初始密码（如 `123456`），并提醒用户首次登录后修改密码。
:::

---

### 场景二：编辑用户信息

1. 在用户列表中找到要编辑的用户
2. 点击「编辑」
3. 修改用户信息（用户名不可修改）
4. 点击「确定」保存

---

### 场景三：重置用户密码

当用户忘记密码时：

1. 找到该用户
2. 点击「重置密码」
3. 确认后密码会重置为默认密码（`123456`）
4. 通知用户新密码，并提醒修改

---

### 场景四：启用/禁用用户

1. 找到要操作的用户
2. 点击「禁用」或「启用」
3. 确认后用户状态变更

禁用后的用户：
- 无法登录系统
- 不会出现在人员选择列表中
- 历史数据保留

---

### 场景五：删除用户

1. 找到要删除的用户
2. 点击「删除」
3. 确认删除

::: warning 删除限制
如果用户有关联的业务数据（如项目、客户等），建议使用「禁用」而不是删除。
:::

---

## 角色管理

### 进入角色管理

点击左侧菜单「系统管理」→「角色管理」

---

### 场景六：创建新角色

1. 点击「新增角色」按钮
2. 填写角色信息：

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 角色编码 | 角色唯一标识，创建后不可修改 | ✅ |
| 角色名称 | 角色显示名称 | ✅ |
| 数据范围 | 全部数据/本部门及下级/本部门/仅本人 | ✅ |
| 排序 | 角色排序顺序 | ❌ |
| 描述 | 角色说明 | ❌ |

3. 点击「确定」保存

---

### 场景七：分配角色权限

1. 找到要配置的角色
2. 点击「分配权限」
3. 在权限树中勾选该角色可以访问的菜单和功能
4. 点击「确定」保存

::: tip 权限继承
勾选父级菜单会自动勾选所有子菜单。取消父级会取消所有子菜单。
:::

---

### 数据范围说明

| 数据范围 | 说明 |
|---------|------|
| 全部数据 | 可以查看系统中所有数据 |
| 本部门及下级 | 可以查看本部门及下级部门的数据 |
| 本部门 | 只能查看本部门的数据 |
| 仅本人 | 只能查看自己创建或负责的数据 |

## 系统配置

### 进入系统配置

点击左侧菜单「系统管理」→「系统配置」

页面标题为「系统配置」，描述为「管理系统配置参数」。

页面分为五个标签页：
- **合同编号配置**：配置合同编号生成规则
- **维护模式**：开启/关闭系统维护模式
- **版本信息**：查看系统版本和运行环境
- **邮件通知**：配置邮件服务和告警通知
- **通用配置**：管理其他系统配置项

---

### 场景十三：配置合同编号规则

合同编号配置用于定义合同编号的生成格式。

1. 切换到「合同编号配置」标签页
2. 设置编号规则：

| 字段 | 说明 |
|------|------|
| 编号前缀 | 用于 {PREFIX} 变量，如「HT」「CONTRACT」 |
| 序号长度 | 序号不足位数前面补0，如设置4则显示0001 |
| 编号规则 | 使用变量组合定义编号格式 |

3. 右侧「编号预览」实时显示生成效果
4. 可选择案件类型查看不同类型的编号效果
5. 点击「保存配置」保存

**推荐规则模板**：
- 点击左下方的推荐模板可快速应用
- 模板包含：标准格式、年度序号、案件类型等多种方案

**支持的变量**：
- 点击变量标签可直接插入到编号规则中
- 常用变量：{PREFIX}、{YEAR}、{MONTH}、{DATE}、{SEQUENCE}、{CASE_TYPE}、{RANDOM} 等

---

### 场景十四：开启维护模式

当需要进行系统维护时，可以开启维护模式阻止普通用户访问。

1. 切换到「维护模式」标签页
2. 输入维护提示信息（如「系统正在维护中，预计维护时间：30分钟，请稍后再试」）
3. 点击「开启维护模式」

::: warning 维护模式注意事项
- 开启维护模式后，只有管理员账户可以访问系统
- 普通用户访问时将看到维护提示信息
- 维护模式下，登录接口仍然可用，允许管理员登录
- 请确保在维护完成后及时关闭维护模式
:::

---

### 场景十五：关闭维护模式

维护完成后：

1. 切换到「维护模式」标签页
2. 点击「关闭维护模式」
3. 系统恢复正常访问

---

### 场景十六：查看版本信息

1. 切换到「版本信息」标签页
2. 查看系统版本和运行环境信息：

| 信息 | 说明 |
|------|------|
| 应用版本 | 当前系统版本号（优先从数据库配置读取） |
| 构建版本 | Maven 构建时的版本号（如与应用版本不同时显示） |
| 构建时间 | 应用打包构建的时间 |
| Git 提交 | 构建时对应的 Git 提交 ID（短格式） |
| 运行环境 | 当前 Spring Profile（dev/test/prod） |
| Java 版本 | Java 运行时版本 |
| Java 供应商 | Java 供应商信息 |
| 操作系统 | 服务器操作系统信息 |
| 服务器时间 | 服务器当前时间 |

::: tip 版本号设置
可以在「通用配置」中修改 sys.version 来设置简单的版本号格式（如 0.4）
:::

---

### 场景十七：配置邮件通知

邮件通知用于发送系统告警和运行报告。

1. 切换到「邮件通知」标签页
2. 查看邮件服务状态（已启用/未启用）
3. 如需启用，在「通用配置」中配置以下参数：

| 配置项 | 说明 |
|--------|------|
| notification.email.enabled | 设为 true 启用邮件服务 |
| notification.email.smtp.host | SMTP 服务器地址 |
| notification.email.smtp.port | SMTP 端口（如 465） |
| notification.email.smtp.username | 发件邮箱地址 |
| notification.email.smtp.password | 邮箱授权码 |
| notification.email.admin.recipients | 告警接收邮箱（多个用逗号分隔） |

**常用 SMTP 配置**：
- QQ邮箱：smtp.qq.com，端口 465 (SSL)，密码使用授权码
- 163邮箱：smtp.163.com，端口 465 (SSL)，密码使用授权码
- Gmail：smtp.gmail.com，端口 587 (TLS)，需开启两步验证

---

### 场景十八：测试邮件配置

1. 在「邮件通知」标签页
2. 输入测试邮箱地址
3. 点击「发送测试邮件」
4. 检查邮箱是否收到测试邮件

---

### 场景十九：测试告警功能

1. 在「邮件通知」标签页
2. 点击对应的测试按钮：
   - 测试登录失败告警
   - 测试账户锁定告警
   - 测试磁盘空间告警
   - 测试备份失败告警
3. 检查管理员邮箱是否收到告警邮件

---

### 场景二十：预览和发送系统报告

1. 在「邮件通知」标签页
2. 点击「预览日报」或「预览周报」查看报告内容
3. 点击「立即发送日报」或「立即发送周报」发送报告

**定时报告配置**：
- notification.report.daily.enabled：启用后每天早上8点自动发送日报
- notification.report.weekly.enabled：启用后每周一早上9点自动发送周报

---

### 场景二十一：修改通用配置

1. 切换到「通用配置」标签页
2. 在配置列表中找到要修改的配置项
3. 点击「编辑」
4. 修改配置值
5. 点击「确定」保存

常见配置项按功能分组：
- **系统基础信息**：sys.name（系统名称）、sys.version（版本号）、sys.copyright（版权信息）
- **登录安全**：sys.login.captcha（验证码）、sys.login.maxAttempts（最大尝试次数）、sys.login.lockDuration（锁定时长）
- **密码配置**：sys.password.minLength（最小长度）、sys.password.complexity（复杂度要求）
- **会话配置**：sys.session.timeout（会话超时时间）
- **上传配置**：sys.upload.maxSize（最大文件大小）、sys.upload.allowTypes（允许的文件类型）

::: warning 谨慎修改
系统配置会影响整个系统的运行，修改前请确认影响范围。
:::

   - 时间范围：按操作时间筛选

---

### 场景二十三：查看日志详情

1. 在日志列表中找到要查看的记录
2. 点击「详情」
3. 查看完整的操作信息：
   - 操作人
   - 操作时间
   - 操作类型
   - 操作模块
   - 请求参数
   - 响应结果
   - IP 地址
   - 浏览器信息

## 外部系统集成

### 进入外部系统集成

点击左侧菜单「系统管理」→「外部集成」

页面标题为「外部系统集成」，描述为「管理档案馆API、AI大模型等外部系统的连接配置」。

页面分为三个标签页：
- **AI大模型**：配置 AI 智能辅助功能
- **档案系统**：配置档案迁移对接
- **其他系统**：其他外部系统集成

---

### 场景二十四：配置 AI 大模型

AI 大模型用于系统中的智能辅助功能，如文书生成、合同审查等。

1. 切换到「AI大模型」标签页
2. 找到要配置的 AI 服务（如 OpenAI、Claude、DeepSeek 等）
3. 点击「配置」
4. 填写配置信息：

| 字段 | 说明 | 必填 |
|------|------|:----:|
| API地址 | AI 服务的 API 地址 | ✅ |
| 认证方式 | API Key / Bearer Token / Basic Auth / OAuth 2.0 | ✅ |
| API Key | API 密钥（留空表示不修改） | ❌ |
| API Secret | API 密钥（部分服务需要） | ❌ |
| 描述 | 配置说明 | ❌ |

5. 点击「确定」保存

---

### 场景二十五：测试 AI 连接

1. 找到已配置的 AI 服务
2. 点击「测试」
3. 系统会尝试连接 AI 服务
4. 测试结果显示：
   - **成功**（绿色）：连接正常
   - **失败**（红色）：连接失败，查看详情了解原因

---

### 场景二十六：启用/禁用 AI 服务

- 点击「启用」：启用该 AI 服务，可在文书制作等功能中使用
- 点击「禁用」：禁用该 AI 服务

::: tip 多模型支持
可以同时启用多个 AI 模型，在使用时可以选择具体使用哪个模型。
:::

---

### 场景二十七：配置档案系统

档案系统配置用于档案迁移功能，将律所档案同步到外部档案管理系统。

1. 切换到「档案系统」标签页
2. 找到要配置的档案系统（如市档案馆、区档案馆）
3. 点击「配置」填写 API 地址和认证信息
4. 点击「测试」验证连接
5. 点击「启用」启用该档案系统

---

## 提成规则配置

### 进入提成规则配置

点击左侧菜单「系统管理」→「提成规则配置」

页面标题为「提成规则配置」，描述为「设置案件收入分配预设方案，律师创建合同时可选择并自定义」。

---

### 场景二十八：查看提成规则

页面显示所有提成规则方案，包括：

| 列名 | 说明 |
|------|------|
| 方案名称 | 规则名称 |
| 律所 | 律所提成比例 |
| 主办律师 | 主办律师提成比例 |
| 协办律师 | 协办律师提成比例 |
| 辅助人员 | 辅助人员提成比例 |
| 案源人 | 案源人提成比例 |
| 可修改 | 律师创建合同时是否可以自定义比例 |
| 默认 | 是否为默认方案 |
| 状态 | 启用/停用 |

::: tip 比例说明
- 比例可以为 0%，表示该角色不参与分配
- 比例之和不强制等于 100%，根据实际情况灵活设置
:::

---

### 场景二十九：新增提成规则

1. 点击「新增方案」按钮
2. 填写规则信息：

| 字段 | 说明 |
|------|------|
| 方案名称 | 如「标准分配方案」 |
| 律所比例 | 律所提成比例（0-100%） |
| 主办律师比例 | 主办律师提成比例 |
| 协办律师比例 | 协办律师提成比例 |
| 辅助人员比例 | 辅助人员提成比例 |
| 案源人比例 | 案源人提成比例 |
| 允许修改 | 律师创建合同时是否可以自定义 |
| 说明 | 方案说明 |

3. 点击「确定」保存

---

### 场景三十：管理提成规则

- **编辑**：点击「编辑」修改规则
- **设为默认**：点击「设为默认」，新建合同时默认使用此规则
- **启用/停用**：点击「启用」或「停用」切换状态
- **删除**：点击「删除」删除规则（默认方案不能删除）

---

### 常见提成场景

| 场景 | 律所 | 主办 | 协办 | 辅助 | 案源 |
|------|:----:|:----:|:----:|:----:|:----:|
| 独立办案 | 30% | 70% | 0% | 0% | 0% |
| 团队协作 | 20% | 40% | 30% | 10% | 0% |
| 律所不提成 | 0% | 70% | 30% | 0% | 0% |

4. 确认后权限立即生效
5. 点击「退出编辑」结束编辑模式

---

### 场景三十三：对比角色权限

## 常见问题

**Q: 用户忘记密码怎么办？**
A: 管理员可以在用户管理中重置用户密码为默认密码。

**Q: 如何批量创建用户？**
A: 目前系统不支持批量导入用户，需要逐个创建。

**Q: 角色权限修改后什么时候生效？**
A: 权限修改后，用户需要重新登录才能生效。

**Q: 可以删除系统内置角色吗？**
A: 系统内置角色（如 ADMIN、DIRECTOR 等）不能删除，只能修改权限。

**Q: 操作日志会保存多久？**
A: 操作日志会长期保存，可以点击「清理历史日志」清理90天前的日志。

**Q: 如何查看某个用户的所有操作？**
A: 在操作日志中，使用「操作人」筛选条件，选择该用户即可。

**Q: AI 大模型配置后多久生效？**
A: 配置保存后立即生效，可以点击「测试」验证连接是否正常。

**Q: 可以同时启用多个 AI 模型吗？**
A: 可以启用多个，在文书制作时可以选择使用哪个模型。

**Q: 提成规则可以修改吗？**
A: 可以。编辑提成规则后，新创建的合同会使用新规则，已有合同不受影响。

**Q: 权限矩阵中如何快速修改权限？**
A: 点击「编辑权限」进入编辑模式，然后点击单元格即可快速切换权限。

**Q: 备份文件存储在哪里？**
A: 备份文件存储在服务器的备份目录中，可以通过「下载」功能下载到本地。

**Q: 恢复备份需要多长时间？**
A: 取决于数据量大小，通常几分钟到几十分钟不等。恢复过程中请勿关闭页面。

**Q: 字典编码可以修改吗？**
A: 字典编码创建后不建议修改，因为代码中可能已经使用了该编码。

**Q: 数据库迁移失败怎么办？**
A: 查看迁移详情中的错误信息，修复问题后可以重新执行。如果无法修复，请联系技术支持。

**Q: 合同编号规则修改后会影响已有合同吗？**
A: 不会。修改编号规则只影响新创建的合同，已有合同的编号保持不变。

**Q: 维护模式下管理员可以正常使用系统吗？**
A: 是的。维护模式只阻止普通用户访问，管理员可以正常登录和使用系统。

**Q: 邮件通知配置后如何测试？**
A: 在「邮件通知」标签页输入测试邮箱地址，点击「发送测试邮件」即可测试。

**Q: 系统报告包含哪些内容？**
A: 日报包含当天的登录统计、操作统计、系统状态等；周报包含一周的汇总数据和趋势分析。

**Q: 如何修改系统显示的版本号？**
A: 在「通用配置」中修改 sys.version 配置项的值即可。
