# 档案管理

案件结案后，需要将相关材料整理归档。档案管理帮助你管理实体档案的入库、借阅和迁移。

## 谁能使用档案管理？

| 功能 | 管理员 | 主任 | 团队负责人 | 财务 | 律师 | 行政 | 实习 |
|------|:-----:|:----:|:--------:|:----:|:----:|:----:|:----:|
| 档案列表 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ |
| 档案借阅 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ |
| 档案迁移 | ✅ | ✅ | ✅ | ❌ | ✅ | ❌ | ❌ |

::: warning 实习律师限制
实习律师无法访问档案管理模块。如需查阅档案，请联系律师协助。
:::

---

## 进入档案管理

点击左侧菜单「档案管理」，可以看到三个子菜单：
- 档案列表：管理所有档案，创建新档案
- 档案借阅：申请和管理借阅记录
- 档案迁移：将档案迁移到外部档案管理系统

---

## 文档与档案的区别

| 类型 | 说明 | 存储位置 | 查看方式 |
|------|------|---------|---------|
| 文档 | 电子文件 | 存储在系统服务器 | 随时在线查看、下载 |
| 档案 | 实体材料（纸质文件） | 存放在档案室 | 需要借阅才能查看 |

档案管理主要管理的是**实体档案**，即纸质文件的存放、借阅和迁移。

## 档案列表

### 进入档案列表

点击左侧菜单「档案管理」→「档案列表」

页面标题为「档案列表」，描述为「管理项目档案」。

### 筛选条件

| 筛选条件 | 说明 |
|----------|------|
| 档案名称 | 输入档案名称关键字 |
| 项目编号 | 输入项目编号 |
| 档案类型 | LITIGATION（诉讼）/NON_LITIGATION（非诉）/CONSULTATION（咨询） |
| 状态 | PENDING（待入库）/PENDING_STORE（待入库审批）/STORED（已入库）/BORROWED（借出）/PENDING_MIGRATE（待迁移审批）/MIGRATED（已迁移） |

### 场景一：案子结了，我要把材料归档

当项目状态变为「已结案」（CLOSED）或「已归档」（ARCHIVED）后，就可以创建档案了。系统提供了**归档向导**，引导你完成整个归档流程。

**创建档案步骤（归档向导）：**

1. 进入「档案管理」→「档案列表」
2. 点击「创建档案」按钮，打开归档向导
3. 按照向导的 4 个步骤完成归档

#### 步骤 1：选择项目

- 从下拉框中选择要归档的项目
- 只有「已结案」（CLOSED）或「已归档」（ARCHIVED）状态的项目才能创建档案
- 支持搜索项目编号或名称

::: tip 快捷入口
你也可以从项目详情页直接点击「归档」按钮，系统会自动跳转到归档向导并选中该项目，自动开始检查归档条件。
:::

#### 步骤 2：检查条件

系统会自动检查项目是否满足归档条件：

- **检查通过**：显示绿色提示「检查通过 - 项目满足归档条件，可以继续」
- **检查未通过**：显示红色提示「检查未通过 - 项目不满足归档条件，请先处理以下问题」

常见的检查项包括：
- 是否有未完成的任务
- 是否有未归还的借阅
- 是否有必要的文档

如果有警告信息，可以选择忽略继续，但建议先处理完再归档。

#### 步骤 3：预览数据

系统会显示将要归档的项目数据统计：

| 统计项 | 说明 |
|--------|------|
| 客户 | 关联的客户数量 |
| 团队成员 | 参与项目的成员数量 |
| 合同 | 关联的合同数量 |
| 收费记录 | 收费记录数量 |
| 工时记录 | 工时记录数量 |
| 文档 | 上传的文档数量 |
| 证据 | 证据材料数量 |
| 审批 | 审批记录数量 |
| 任务 | 任务数量 |

你可以选择要包含在档案中的数据源：
- 勾选需要归档的数据类型
- 必填项（带红色「必填」标签）无法取消勾选

#### 步骤 4：确认创建

确认档案信息无误后，点击「确认创建」：
- 系统会自动生成档案编号
- 项目状态会自动更新为「已归档」（ARCHIVED）
- 档案初始状态为「待入库」（PENDING）

---

### 场景二：档案整理好了，要入库

创建档案后，需要把实体材料放到档案室，并在系统中登记。

**入库流程：**

1. 在档案列表中找到状态为「待入库」（PENDING）的档案卡片
2. 点击「提交审批」按钮
3. 确认后，档案状态变为「待入库审批」（PENDING_STORE）
4. 等待档案管理员审批
5. 审批通过后，档案状态变为「已入库」（STORED）

**审批入库（档案管理员）：**

1. 在档案列表中找到状态为「待入库审批」（PENDING_STORE）的档案
2. 点击「审批」按钮
3. 在弹窗中查看档案信息
4. 选择审批结果（通过/拒绝）
5. 填写审批意见
6. 点击「确认」

---

### 场景三：下载卷宗封面

系统会自动生成标准化的卷宗封面（PDF格式），用于打印贴在实体档案盒上。

**下载封面步骤：**

1. 在档案列表中找到档案卡片
2. 点击卡片下方的「封面」按钮
3. 在下拉菜单中选择「下载封面」
4. 系统会下载 PDF 格式的卷宗封面

**封面包含的信息：**
- 业务档案卷宗标题
- 案件类型（如民事诉讼类、刑事诉讼类等）
- 项目名称和编号
- 档案编号
- 客户名称
- 主办律师
- 结案日期和归档日期
- 档案状态

::: tip 封面颜色标识
不同案件类型的封面有不同的颜色标识条：
- 🔴 刑事案件（CRIMINAL）：红色
- 🔵 民事案件（CIVIL）：蓝色
- 🟢 行政案件（ADMINISTRATIVE）：绿色
- 🟠 破产案件（BANKRUPTCY）：橙色
- 🟣 知识产权（IP）：紫色
- 🔵 仲裁案件（ARBITRATION）：青色
- 🟤 执行案件（ENFORCEMENT）：棕色
- 🟢 法律顾问（LEGAL_COUNSEL）：深青色
- 🟠 专项服务（SPECIAL_SERVICE）：深橙色
:::

### 场景四：重新生成卷宗封面

如果档案信息有更新，可以重新生成封面：

1. 在档案列表中找到档案卡片
2. 点击卡片下方的「封面」按钮
3. 在下拉菜单中选择「重新生成」
4. 确认后系统会重新生成封面

## 档案借阅

进入「档案管理」→「档案借阅」

页面标题为「档案借阅」，描述为「管理档案借阅记录」。

页面顶部有四个标签页：
- **借阅中**：当前正在借阅的档案（状态为BORROWING）
- **已归还**：已经归还的借阅记录（状态为RETURNED）
- **逾期未还**：超过预计归还日期还没还的
- **全部**：所有借阅记录

### 场景五：我要借阅一份档案

**申请借阅步骤：**

1. 进入「档案借阅」页面
2. 点击「申请借阅」按钮
3. 填写借阅信息：

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 选择档案 | 从下拉框选择要借阅的档案（支持搜索） | ✅ |
| 预计归还日期 | 计划什么时候还 | ✅ |
| 借阅事由 | 为什么要借阅 | ❌ |

4. 点击「确定」提交申请

::: info 借阅流程
1. **待审批**（PENDING）：等待档案管理员审批
2. **已通过**（APPROVED）：审批通过，可以去档案室取档案了
3. **借阅中**（BORROWING）：档案管理员确认借出后的状态
:::

---

### 场景六：我是档案管理员，要审批借阅申请

**审批步骤：**

1. 在「全部」标签页中找到状态为「待审批」（PENDING）的记录
2. 点击「通过」或「拒绝」
   - **通过**：同意借阅申请
   - **拒绝**：需要输入拒绝原因

**确认借出：**
审批通过后，当借阅人来取档案时，点击「确认借出」，档案状态变为「借阅中」（BORROWING）。

---

### 场景七：档案用完了，要归还

**归还步骤：**

1. 在「借阅中」标签页找到要归还的记录
2. 点击「归还」按钮
3. 填写归还信息：
   - **归还日期**：选择实际归还日期（必填）
   - **档案状况**：描述档案的状况（可选）
4. 点击「确定」完成归还

::: warning 逾期提醒
超过预计归还日期还没归还，记录会出现在「逾期未还」标签页。请及时归还档案！
:::

---

|---------|---------|------|------|
| PENDING | 待审批 | 橙色 | 等待档案管理员审批 |
| APPROVED | 已通过 | 蓝色 | 审批通过，可以取档案 |

## 档案迁移

进入「档案管理」→「档案迁移」

页面标题为「档案迁移」，描述为「将档案迁移到外部档案管理系统」。

::: info 什么是档案迁移？
档案迁移是将本系统中的档案转移到外部档案管理系统（如市档案馆、区档案馆等）的过程。迁移后，可选择删除本系统中的实体文件以节省存储空间，但档案信息将保留用于查询。
:::

页面顶部有四个标签页：
- **可迁移档案**：已入库的档案（状态为STORED），可以申请迁移
- **待审批**：已提交迁移申请，等待审批（状态为PENDING_MIGRATE）
- **已迁移**：已经迁移到外部系统的档案（状态为MIGRATED）
- **全部**：所有档案

### 场景八：档案需要迁移到外部系统

**申请迁移步骤：**

1. 进入「档案迁移」页面
2. 在「可迁移档案」标签页找到要迁移的档案
3. 点击「申请迁移」按钮
4. 填写迁移信息：

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 迁移目标 | 选择迁移目的地 | ✅ |
| 迁移原因 | 说明为什么要迁移 | ✅ |

**迁移目标选项：**
- 市档案馆
- 区档案馆
- 第三方档案管理系统
- 其他

5. 点击「确定」提交申请

---

### 场景九：我是档案管理员，要审批迁移申请

**审批步骤：**

1. 在「待审批」标签页找到迁移申请
2. 点击「审批」按钮
3. 查看档案信息和迁移原因
4. 选择审批结果（通过/拒绝）
5. 填写审批意见
6. 如果通过，可以选择是否删除本系统中的实体文件：
   - 勾选「迁移后删除本系统中的实体文件（保留档案信息）」可以节省存储空间
   - 档案信息会保留，只是删除文件
7. 点击「确认」

::: warning 文件删除提醒
如果选择删除文件，迁移后将无法在本系统中查看或下载档案文件。请确保已在目标系统中妥善保存。
:::

## 常见问题

**Q: 借阅申请被拒绝了怎么办？**
A: 查看拒绝原因，根据原因调整后重新申请。

**Q: 借阅档案有时间限制吗？**
A: 有的。申请时需要填写预计归还日期，超过日期未归还会显示为「逾期」。

**Q: 档案迁移后还能在系统中查看吗？**
A: 档案信息会保留，可以查看基本信息和迁移记录。但如果选择了删除文件，则无法查看或下载文件内容。

**Q: 什么情况下需要迁移档案？**
A: 常见情况包括：
- 保管期限到期，需要移交档案馆
- 系统存储空间不足
- 按照档案管理规定需要移交

**Q: 只有已入库的档案才能迁移吗？**
A: 是的。档案必须先完成入库流程，状态为「已入库」（STORED）才能申请迁移。

**Q: 如何从项目详情页直接归档？**
A: 在项目详情页，如果项目状态为「已结案」（CLOSED），会显示「归档」按钮。点击后会自动跳转到档案列表页面并打开归档向导，自动选中该项目并开始检查归档条件。
