# 客户管理

客户管理模块用于管理律所的客户信息，包括客户录入、利冲审查、案源管理。

## 谁能使用？

| 功能     | 管理员 | 主任 | 团队负责人 |   财务   | 律师 |   行政   | 实习 |
| -------- | :----: | :--: | :--------: | :------: | :--: | :------: | :--: |
| 客户列表 |   ✅   |  ✅  |     ✅     | ✅(查看) |  ✅  | ✅(查看) |  ❌  |
| 利冲审查 |   ✅   |  ✅  |     ✅     |    ❌    |  ✅  |    ❌    |  ❌  |
| 案源管理 |   ✅   |  ✅  |     ✅     |    ❌    |  ✅  |    ❌    |  ❌  |

## 客户列表

**菜单位置**：客户管理 → 客户列表

### 查询筛选

| 筛选项   | 说明                                                            |
| -------- | --------------------------------------------------------------- |
| 创建年份 | 按创建年份筛选，支持选择"全部年份"或最近5年（默认显示当前年份） |
| 客户名称 | 模糊搜索客户名称（支持输入后按回车或点击查询按钮）              |
| 客户类型 | 企业客户/个人客户/政府机关/其他（支持清空选择）                 |
| 客户级别 | 重要客户(A)/普通客户(B)/一般客户(C)（支持清空选择）             |
| 状态     | 潜在/正式/休眠/黑名单（支持清空选择）                           |

### 客户列表字段

| 列名     | 说明                                               |
| -------- | -------------------------------------------------- |
| 客户编号 | 系统自动生成（clientNo）                           |
| 客户名称 | 企业名称或个人姓名                                 |
| 客户类型 | 企业/个人/政府/其他                                |
| 联系人   | 联系人姓名                                         |
| 联系电话 | 联系电话                                           |
| 客户级别 | 重要客户(A)/普通客户(B)/一般客户(C)                |
| 客户分类 | 重要客户(VIP)/普通客户(NORMAL)/潜在客户(POTENTIAL) |
| 状态     | 当前状态                                           |
| 案源人   | 引入该客户的律师                                   |
| 负责律师 | 负责该客户的律师                                   |
| 创建时间 | 录入时间                                           |

### 新增客户

1. 点击「新增客户」按钮
2. 填写客户信息：

**OCR 智能识别填充**：

::: warning OCR功能状态当前版本中OCR功能暂时禁用，本地识别服务优化中，后续版本将重新开放。:::

新增客户时可以使用 OCR 功能自动识别证件信息：

- **营业执照识别**：上传营业执照图片，自动填充企业名称、统一社会信用代码、法定代表人，并自动设置客户类型为"企业客户"
- **身份证识别**：上传身份证正面图片，自动填充姓名、身份证号，并自动设置客户类型为"个人客户"

::: tip OCR识别提示如果OCR识别质量较低（图片不清晰或不完整），系统会显示原始文本供手动提取信息。:::

**基本信息**：

| 字段             | 必填 | 说明                            |
| ---------------- | :--: | ------------------------------- |
| 客户名称         |  ✅  | 企业全称或个人姓名              |
| 客户类型         |  ✅  | 企业客户/个人客户/政府机关/其他 |
| 统一社会信用代码 |  -   | 企业客户填写（18位）            |
| 身份证号         |  -   | 个人客户填写                    |
| 法定代表人       |  -   | 企业客户填写                    |

**联系信息**：

| 字段     | 必填 | 说明         |
| -------- | :--: | ------------ |
| 联系人   |  -   | 联系人姓名   |
| 联系电话 |  -   | 联系电话     |
| 联系邮箱 |  -   | 电子邮箱     |
| 所属行业 |  -   | 客户所属行业 |

**业务信息**：

| 字段 | 必填 | 说明 |
| --- | :-: | --- |
| 客户级别 | - | 重要客户(A)/普通客户(B)/一般客户(C)，默认B |
| 客户分类 | - | 重要客户(VIP)/普通客户(NORMAL)/潜在客户(POTENTIAL)，默认NORMAL |
| 案源人 | - | 引入客户的律师（支持按部门筛选） |
| 负责律师 | - | 负责该客户的律师（支持按部门筛选） |

**利益冲突检索**（新增时必填）：

| 字段       | 必填 | 说明                         |
| ---------- | :--: | ---------------------------- |
| 对方当事人 |  -   | 如有案件时填写对方当事人名称 |

3. 如果填写了对方当事人，需要点击「🔍 利冲检索」按钮进行利益冲突检查
4. 系统会自动匹配现有客户，返回可能的冲突信息
5. 根据风险级别显示不同提示：
   - **高风险（红色）**：可能存在利益冲突，需仔细核对
   - **中风险（橙色）**：发现相似客户，请确认是否为同一人/公司
   - **低风险（蓝色）**：发现名称相近的客户，请核对后继续
   - **无冲突（绿色）**：未检测到利益冲突，可以正常创建
6. 点击「确定」保存

### 编辑客户

1. 点击客户行的「编辑」按钮
2. 修改客户信息
3. 点击「确定」保存

### 删除客户

1. 点击「删除」按钮
2. 确认删除

::: warning 注意已关联项目或合同的客户不能删除。:::

### 批量删除

1. 勾选要删除的客户（使用表格左侧复选框）
2. 点击「批量删除 (N)」按钮（N为选中数量）
3. 确认删除

### 转为正式客户

潜在客户可以转为正式客户：

1. 找到状态为「潜在」(POTENTIAL) 的客户
2. 点击「转正式」按钮
3. 确认转正

### 客户状态说明

| 状态      | 颜色 | 说明       |
| --------- | ---- | ---------- |
| POTENTIAL | 橙色 | 潜在客户   |
| ACTIVE    | 绿色 | 正式客户   |
| INACTIVE  | 灰色 | 休眠客户   |
| BLACKLIST | 红色 | 黑名单客户 |

## 利冲审查

**菜单位置**：客户管理 → 利冲审查

利益冲突审查是接案前的必要步骤，确保新案件与现有案件没有利益冲突。

### 查询筛选

| 筛选项   | 说明                                            |
| -------- | ----------------------------------------------- |
| 客户名称 | 模糊搜索客户名称                                |
| 状态     | 待检查/已通过/存在冲突/豁免待审批/已豁免/已拒绝 |

### 利冲审查列表字段

| 列名       | 说明                                  |
| ---------- | ------------------------------------- |
| 审查编号   | 系统自动生成（checkNo）               |
| 客户名称   | 申请审查的客户名称                    |
| 对方当事人 | 对方当事人名称                        |
| 项目名称   | 关联的项目名称（可选）                |
| 审查类型   | 新客户(NEW_CLIENT)/新项目(NEW_MATTER) |
| 状态       | 当前审查状态                          |
| 结果       | 审查结果                              |
| 申请人     | 申请人姓名                            |
| 申请时间   | 申请时间                              |

### 申请利冲审查

1. 点击「申请利冲审查」按钮
2. 填写审查信息：

**OCR 智能识别对方当事人**（如已启用）：

- **识别身份证**：上传对方身份证，自动识别姓名填充到对方当事人
- **识别营业执照**：上传对方营业执照，自动识别企业名称填充到对方当事人

| 字段       | 必填 | 说明                        |
| ---------- | :--: | --------------------------- |
| 审查类型   |  ✅  | 新客户/新项目               |
| 客户名称   |  ✅  | 我方客户名称                |
| 对方当事人 |  ✅  | 对方当事人名称（可OCR识别） |
| 项目名称   |  -   | 关联的项目名称              |
| 备注       |  -   | 其他说明                    |

3. 点击「确定」提交申请

### 审查状态

| 状态              | 颜色   | 说明       |
| ----------------- | ------ | ---------- |
| PENDING           | 橙色   | 待检查     |
| CHECKING          | 蓝色   | 检查中     |
| PASSED            | 绿色   | 已通过     |
| CONFLICT          | 红色   | 存在冲突   |
| EXEMPTION_PENDING | 紫色   | 豁免待审批 |
| WAIVED            | 青色   | 已豁免     |
| REJECTED          | 火山色 | 已拒绝     |

### 审查结果

| 结果        | 颜色 | 说明     |
| ----------- | ---- | -------- |
| NO_CONFLICT | 绿色 | 无冲突   |
| CONFLICT    | 红色 | 存在冲突 |
| WAIVER      | 蓝色 | 已豁免   |

### 申请豁免

当利冲检查发现存在冲突时，如确需继续代理，可以申请豁免：

1. 找到状态为「存在冲突」(CONFLICT) 的记录
2. 点击「申请豁免」按钮
3. 填写豁免理由（必填）
4. 提交豁免申请，等待审批

### 审批豁免

豁免申请由管理员或主任审批：

1. 找到状态为「豁免待审批」(EXEMPTION_PENDING) 的记录
2. 点击「批准」或「拒绝」按钮
3. 如拒绝需输入拒绝原因

## 常见问题

**Q: 客户编号是怎么生成的？**

客户编号由系统自动生成，格式为 `C + 年份 + 序号`，如 `C2026010001`。

**Q: 已删除的客户可以恢复吗？**

删除的客户不能恢复，请谨慎操作。如果需要隐藏客户，可以将状态改为「休眠」。

**Q: 利冲检索会检查哪些内容？**

系统会检查客户名称、身份证号、企业信用代码与现有客户和项目的对方当事人是否匹配。

**Q: 一个客户可以有多个联系人吗？**

目前系统只支持一个主要联系人。如果有多个联系人，可以在备注中说明。
