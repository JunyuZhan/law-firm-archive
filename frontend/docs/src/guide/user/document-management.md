# 文档管理

文档管理模块用于管理项目相关的各类文档，支持按项目类型自动生成标准化文件夹结构，并提供文书模板、印章管理、用印申请等功能。

## 谁能使用文档管理？

| 功能 | ADMIN | DIRECTOR | PARTNER | 财务 | 主办律师 | 协办律师 | 行政 | 实习 |
|------|:-----:|:--------:|:-------:|:----:|:-------:|:-------:|:----:|:----:|
| 文书列表 | ✅ | ✅ | ❌ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 文书模板 | ✅ | ✅ | ❌ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 印章管理 | ✅ | ✅ | ❌ | ✅ | ✅ | ✅ | ✅ | ❌ |
| 用印申请 | ✅ | ✅ | ❌ | ✅ | ✅ | ✅ | ✅ | ❌ |

::: warning 实习律师限制
实习律师可以查看文书列表和模板，但不能管理印章和申请用印。
:::

## 项目文档管理

### 进入项目文档管理

点击左侧菜单「文档管理」→「项目文档」

---

### 场景一：选择项目查看文档

1. 在页面顶部的筛选区域：
   - 选择年度：按年份筛选项目
   - 选择项目类型：民事/刑事/行政/法律顾问等
   - 选择项目状态：进行中/已结案等
2. 在「选择项目」下拉框中选择具体项目
3. 选择后，页面显示该项目的文档管理界面

---

### 场景三：上传文档

1. 选择项目后，点击「上传文档」按钮
2. 填写文档信息：

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 文档标题 | 文档的名称 | ✅ |
| 存储位置 | 选择文件夹（分类） | ✅ |
| 安全级别 | 公开/内部/机密/绝密（默认：内部） | ❌ |
| 文档阶段 | 文档所属的项目阶段 | ❌ |
| 标签 | 文档标签，多个标签用逗号分隔 | ❌ |
| 描述 | 文档说明 | ❌ |

3. 选择文件：
   - 点击上传区域或拖拽文件
   - 支持图片、PDF、Word、Excel、PPT等格式
   - 单个文件大小不能超过 100MB

4. 点击「确定」完成上传

::: tip 文档编号
系统会自动为每个文档生成唯一的文档编号，格式为：DOC + 日期（YYMMDD）+ 4位随机码。例如：DOC240105A1B2
:::

::: tip 安全级别说明
- **公开**：所有用户都可以查看
- **内部**：律所内部人员可以查看（默认）
- **机密**：仅授权人员可以查看
- **绝密**：仅特定人员可以查看
:::

### 场景七：文档版本管理

系统支持文档版本管理，可以保留文档的历史版本，方便追溯和恢复。

#### 查看版本历史

1. 找到文档，点击「操作」→「版本历史」
2. 弹窗显示该文档的所有历史版本，包括：
   - 版本号（v1, v2, v3...）
   - 上传时间
   - 上传人
   - 变更说明
   - 文件大小
3. 可以查看每个版本的详细信息
4. 可以下载历史版本

#### 上传新版本

当文档需要更新时，可以上传新版本而不是替换原文档：

1. 找到文档，点击「操作」→「上传新版本」
2. 选择新版本文件
3. 填写变更说明（可选，说明本次更新的内容）
4. 点击「确定」上传

上传后，新版本成为当前版本，旧版本保留在版本历史中。

::: tip 版本管理优势
- 保留所有历史版本，不会丢失旧版本
- 可以对比不同版本的差异
- 支持回退到历史版本
:::

#### 版本回退

如果需要恢复到之前的某个版本：

1. 打开文档的版本历史
2. 找到要回退的目标版本
3. 点击「回退到此版本」
4. 确认后，系统会创建一个新版本（基于目标版本的内容）

::: warning 版本回退说明
- 只能回退到更早的版本，不能回退到更新的版本
- 回退会创建一个新版本，不会删除现有版本
- 回退后的新版本会继承目标版本的文件内容
:::

### 场景八：编辑文档信息

1. 点击「操作」→「编辑」
2. 可以修改以下信息：
   - 文档标题
   - 文档分类（文件夹）
   - 安全级别
   - 文档阶段
   - 标签
   - 描述
3. 点击「确定」保存

::: tip 编辑限制
- 文档的文件内容需要通过「上传新版本」来更新
- 编辑文档信息不会创建新版本
:::

### 场景九：归档文档

对于不再使用的文档，可以归档保存：

1. 找到要归档的文档
2. 点击「操作」→「归档」
3. 确认归档

归档后的文档：
- 状态变为「已归档」
- 仍然可以查看和下载
- 不会在常规列表中显示（可通过筛选查看）

::: tip 归档用途
归档功能用于管理已完成项目的文档，保持文档列表的整洁，同时保留历史记录。
:::

---

### 场景十：查看文档访问日志

系统会记录文档的访问情况，包括查看和下载记录：

1. 找到文档，点击「操作」→「访问日志」
2. 弹窗显示该文档的访问记录，包括：
   - 访问时间
   - 访问人
   - 操作类型（查看/下载）
   - IP地址
   - 用户代理信息

::: tip 访问日志用途
- 追踪文档的使用情况
- 了解哪些人查看了文档
- 用于文档审计和安全监控
:::

### 场景十一：文档审计统计

系统提供文档访问的统计分析功能（需要相应权限）：

1. 在文档管理页面，点击「审计统计」
2. 设置统计条件：
   - 时间范围
   - 文档筛选
   - 用户筛选
   - 操作类型筛选
3. 查看统计结果：
   - **按用户统计**：各用户的访问次数
   - **按文档统计**：各文档的访问次数
   - **按操作类型统计**：查看和下载的次数分布
   - **按时间统计**：访问趋势图

::: tip 审计统计权限
文档审计统计功能需要「文档审计查看」权限，通常只有管理员可以使用。
:::

### 场景十二：删除文档

1. 点击「操作」→「删除」
2. 确认删除

删除后的文档：
- 状态变为「已删除」
- 不会在常规列表中显示
- 可以通过筛选查看已删除的文档

::: warning 删除说明
- 删除操作会标记文档为已删除状态，不会立即物理删除文件
- 已删除的文档可以通过筛选查看，但无法正常使用
- 如需恢复，请联系系统管理员
:::

## 文档分类管理

文档分类用于组织和管理文档，支持树形结构，可以创建多级分类。

### 查看文档分类

1. 在文档管理页面，左侧显示文档分类树
2. 点击分类可以筛选该分类下的文档
3. 分类树支持展开和折叠

### 创建文档分类

1. 在文档分类区域，点击「新建分类」
2. 填写分类信息：
   - 分类名称（必填）
   - 父分类（可选，选择后创建为子分类）
   - 描述（可选）
3. 点击「确定」创建

::: tip 分类层级
- 可以创建多级分类，形成树形结构
- 建议分类层级不超过3级，便于管理
:::

### 编辑文档分类

1. 右键点击分类，选择「编辑」
2. 修改分类名称或描述
3. 可以调整分类的排序顺序
4. 点击「保存」完成编辑

### 删除文档分类

1. 右键点击分类，选择「删除」
2. 确认删除

::: warning 删除分类
- 删除分类前，需要先移动或删除该分类下的所有文档
- 删除分类不会删除分类下的文档，但文档会变为未分类状态
:::

## 文书制作

### 进入文书制作

点击左侧菜单「文档管理」→「文书制作」

页面标题为「文书制作」，描述为「使用模板或 AI 快速制作法律文书」。

文书制作支持两种模式：
- **模板制作**：选择模板 → 选择项目 → 填写内容 → 预览生成
- **AI 智能生成**：描述需求 → 选择项目 → AI 生成 → 保存

---

### 场景一：使用模板制作文书

1. 进入「文书制作」页面
2. 选择「📄 模板制作」模式（默认）
3. **步骤1 - 选择模板**：
   - 可按业务类型筛选（诉讼文书/非诉文书/合同文书/函件文书/其他）
   - 点击模板卡片选择，选中后显示蓝色边框和「已选择」标签
   - 卡片显示模板名称、描述、类型和使用次数
4. 点击「下一步」

5. **步骤2 - 关联项目**：
   - 选择要关联的项目（文书将保存到项目卷宗）
   - 或开启「个人文书」开关，文书保存到「我的文书」
   - 如关联项目，可选择保存到具体的卷宗目录
6. 点击「下一步」

7. **步骤3 - 填写内容**：
   - 输入文件名称
   - 填写模板变量（如委托人姓名、案由等）
   - 系统会自动识别模板中的变量
8. 点击「下一步」

9. **步骤4 - 预览生成**：
   - 左侧显示文书预览效果
   - 右侧显示生成信息（模板、保存位置、文件名）
   - 确认无误后点击「生成文书」

::: tip 文书保存位置
- 关联项目：保存到项目卷宗
- 不关联项目：保存到「我的文书」中查看
:::

---

### 场景二：使用 AI 智能生成文书

1. 进入「文书制作」页面
2. 选择「🤖 AI 智能生成」模式
3. 如果 AI 服务未配置，会显示警告提示

4. **步骤1 - 描述需求**：
   - 选择 AI 大模型（如有多个可用）
   - 选择文书类型：起诉状/答辩状/上诉状/法律意见书/律师函/合同/授权委托书/代理词/辩护词/其他
   - 选择语气风格：正式严谨/温和友好/强硬有力/简洁明了
   - 详细描述文书需求（至少10个字）
   - 可填写补充信息（相关法律条文、证据情况等）
   - 可指定文件名称（留空自动生成）
5. 点击「下一步」

6. **步骤2 - 关联项目 & 收集信息**：
   - 选择关联项目或开启「个人文书」模式
   - **一键收集项目信息**（重要功能）：
     - 展开「选择收集内容」面板
     - 勾选要收集的信息类型：项目基本信息、客户/当事人信息、参与律师信息、相关文档
     - 可开启「提取文档内容」提取 Word、PDF、图片（OCR）的文本
     - 选择要包含的文档
     - 点击「🔍 收集选中信息」
   - **数据脱敏**（推荐用于云端大模型）：
     - 开启后敏感信息会脱敏后发送给 AI
     - 系统记录脱敏映射，生成后可还原
7. 点击「下一步」

8. **步骤3 - 生成预览**：
   - AI 自动生成文书内容
   - 如开启了脱敏，可点击「🔓 脱敏还原」恢复真实信息
   - 可点击「⚡ 重新生成」重新生成
   - 确认无误后点击「💾 保存文书」

::: tip AI 模型选择
- 🟢 OpenAI、🟣 Claude、🔵 DeepSeek 等为云端模型
- 🏠 本地模型（如 Ollama）数据不会上传云端，适合处理敏感信息
:::

::: warning 脱敏还原
使用云端 AI 时建议开启脱敏功能。AI 生成的内容中敏感信息会显示为脱敏格式，点击「脱敏还原」可恢复为真实信息。
:::

## 文书模板

### 进入文书模板

点击左侧菜单「文档管理」→「文书模板」

页面标题为「文书模板」，描述为「管理文书模板库，支持变量替换自动生成文书」。

---

### 场景十：查看模板库

1. 进入文书模板页面
2. 可以按以下条件筛选：

| 筛选条件 | 说明 |
|---------|------|
| 模板名称 | 输入关键词搜索 |
| 模板类型 | Word文档/Excel表格/PDF文档/富文本 |

4. 表格显示所有可用的模板

---

### 模板列表字段说明

| 列名 | 说明 |
|------|------|
| 模板名称 | 模板的名称（点击可预览），下方显示描述 |
| 模板类型 | Word（蓝色）/Excel（绿色）/PDF（红色）/富文本（紫色） |
| 适用业务 | 诉讼案件/非诉项目/法律顾问/刑事辩护/通用 |
| 创建人 | 创建模板的人 |
| 创建时间 | 模板创建的时间 |
| 使用次数 | 模板被使用的次数（蓝色高亮） |
| 状态 | 启用（绿色）/停用（灰色） |

---

### 场景十一：预览模板

1. 点击模板名称，或点击「预览」链接
2. 弹窗显示模板预览，包含三个标签页：

**预览效果标签页**
- 显示模板的渲染效果
- 变量会显示为占位符格式（如 `${clientName}`）
- 富文本模板会渲染为HTML格式

**模板信息标签页**
- 显示模板的详细信息：名称、类型、适用业务、状态
- 显示创建人、创建时间、使用次数
- 显示描述说明

## 用印申请

### 进入用印申请

点击左侧菜单「文档管理」→「用印申请」

页面分为多个标签页（根据权限显示）：
- **我的申请**：我提交的用印申请（所有人可见）
- **待审批**：需要我审批的申请（有审批权限可见）
- **保管人待办理**：审批通过，等待保管人登记用印的申请（印章保管人可见）
- **保管人已办理**：保管人已登记用印的申请（印章保管人可见）
- **全部申请**：所有用印申请（有管理权限可见）

---

### 场景十七：提交用印申请

1. 点击「新建申请」
2. 填写申请信息：

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 印章 | 选择要使用的印章 | ✅ |
| 用印文件 | 需要盖章的文件名称 | ✅ |
| 文件类型 | 合同/协议/函件/证明/其他 | ✅ |
| 审批人 | 选择审批人（必填） | ✅ |
| 关联项目 | 选择关联的项目（可选） | ❌ |
| 用印目的 | 说明用印原因 | ❌ |
| 用印份数 | 需要盖章的份数（默认为1） | ❌ |
| 预计用印日期 | 计划用印的日期 | ❌ |

3. 点击「确定」提交

::: tip 审批人选择
申请时必须选择审批人。系统会根据你的角色和权限显示可选的审批人列表。
:::

::: tip 关联项目
如果用印文件与某个项目相关，建议关联项目，方便后续追溯。
:::

---

### 场景十八：审批用印申请

有两种方式审批用印申请：

**方式一：在用印申请页面审批**

1. 切换到「待审批」标签页
2. 找到待审批的申请，点击「审批」
3. 查看申请详情：
   - 申请编号
   - 印章名称
   - 用印文件
   - 用印目的
   - 用印份数
4. 填写审批意见（可选）
5. 选择审批结果：
   - **通过**：同意用印申请
   - **拒绝**：拒绝申请，需要填写拒绝原因
6. 点击「确定」提交审批

**方式二：在审批中心统一审批（推荐）**

1. 点击页面上的「前往审批中心统一处理」链接
2. 在审批中心可以看到所有待审批事项（包括用印申请）
3. 支持批量审批操作
4. 审批流程与方式一相同

::: warning 审批权限
用印申请需要具有审批权限的用户审批。审批人由申请人在提交时选择。
:::

::: tip 审批中心优势
建议在审批中心统一处理所有待审批事项，支持批量操作，提高效率。
:::

---

### 场景十九：登记用印（保管人操作）

审批通过后，印章保管人需要登记实际用印：

1. 切换到「保管人待办理」标签页（只有印章保管人能看到此标签）
2. 找到状态为「已通过」的申请
3. 点击「登记用印」
4. 填写用印备注（可选）
5. 点击「确定」

登记后，申请状态变为「已用印」，流程完成。

::: warning 保管人权限
只有被设置为印章保管人的用户才能看到「保管人待办理」和「保管人已办理」标签页，并进行登记用印操作。
:::

---

### 场景二十：取消用印申请

如果不需要用印了：

1. 在「我的申请」标签页中，找到状态为「待审批」的申请
2. 点击「取消」
3. 确认后申请被取消

::: warning 取消限制
只有「待审批」状态的申请可以取消。已通过或已用印的申请不能取消。
:::

## 常见问题

**Q: 上传的文档存储在哪里？**
A: 文档存储在系统配置的文件服务器上，按项目和文件夹分类存储。

**Q: 可以批量上传文档吗？**
A: 可以，在上传弹窗中可以同时选择多个文件。

**Q: 模板中的变量怎么写？**
A: 使用 `${变量名}` 格式，如 `${clientName}`、`${matterNo}`。系统提供了预设变量，点击变量标签即可插入。

**Q: 模板变量会自动替换吗？**
A: 是的，生成文书时系统会自动将变量替换为实际数据。例如 `${clientName}` 会替换为委托人的真实姓名。

**Q: 可以自定义变量吗？**
A: 目前只支持系统预设的变量。如需新增变量，请联系系统管理员。

**Q: 富文本模板和其他类型有什么区别？**
A: 富文本模板使用可视化编辑器，支持格式设置、插入表格等功能。其他类型使用纯文本编辑器，适合需要精确控制格式的场景。

**Q: 预设模板可以修改吗？**
A: 预设模板只是快速加载的起点，加载后可以自由修改内容。

**Q: 用印申请被拒绝后可以重新申请吗？**
A: 可以，重新提交一个新的用印申请即可。

**Q: 印章使用次数是怎么统计的？**
A: 每次用印登记完成后，系统自动累加使用次数。

**Q: 文档编号可以修改吗？**
A: 不可以。文档编号由系统自动生成，用于唯一标识文档，不能手动修改。

**Q: 文档版本有数量限制吗？**
A: 理论上没有限制，但建议定期清理不需要的旧版本，保持版本历史的整洁。

**Q: 如何查看文档的访问记录？**
A: 在文档的操作菜单中，点击「访问日志」可以查看该文档的所有访问记录。

**Q: 归档和删除有什么区别？**
A: 归档是标记文档为已归档状态，文档仍然可以查看；删除是标记文档为已删除状态，文档不再正常显示。归档更适合已完成项目的文档管理。

**Q: 可以批量操作文档吗？**
A: 可以。在文档列表中，可以勾选多个文档进行批量操作，如批量归档、批量删除等。

**Q: 文档的安全级别如何设置？**
A: 上传文档时可以选择安全级别，也可以后续编辑文档信息时修改。安全级别影响文档的可见性。

**Q: 文档访问日志会保存多久？**
A: 文档访问日志会长期保存，用于审计和追溯。具体保存期限由系统配置决定。

**Q: 如何恢复已删除的文档？**
A: 已删除的文档可以通过筛选查看，但无法直接恢复。如需恢复，请联系系统管理员。

**Q: 文档分类可以嵌套多少层？**
A: 理论上没有限制，但建议不超过3层，便于管理和查找。

**Q: AI 文书生成需要什么条件？**
A: 需要在「系统管理」→「外部集成」中配置并启用 AI 大模型服务。

**Q: AI 生成的文书质量如何？**
A: AI 生成的文书可以作为初稿参考，建议律师审核修改后使用。提供越详细的需求描述和项目信息，生成质量越高。

**Q: 什么是数据脱敏？**
A: 脱敏是将敏感信息（如姓名、身份证号、电话等）替换为占位符后发送给 AI，保护隐私。生成后可以一键还原为真实信息。

**Q: 本地 AI 模型和云端模型有什么区别？**
A: 本地模型（如 Ollama）数据不会上传到云端，适合处理敏感信息；云端模型（如 OpenAI、Claude）通常能力更强，但数据会发送到云端。
