# Gemini SVG 插画生成提示词

## 🎨 提示词（Prompt）

````
请生成一个适合法律行业管理系统的登录页面SVG插画代码。

**设计要求：**
1. **风格**：现代扁平化设计，等距投影（isometric）风格，与现有代码风格一致
2. **尺寸**：viewBox="0 0 800 800"，适合登录页面左侧展示
3. **主题元素**：法律行业相关，包含以下元素：
   - 法律书籍/法典（3D等距投影风格）
   - 天平（正义的象征，等距投影）
   - 文件夹/卷宗（堆叠或展开状态）
   - 律师/法官人物形象（简化的等距投影小人）
   - 合同文件/法律文档
   - 数据图表（柱状图或折线图，表示数据分析）
   - 云朵图标（表示云端存储）
   - 发光的圆形底座或平台

4. **配色方案**：
   - 主色：深蓝色系（#2563EB, #1E40AF）表示专业、信任
   - 辅助色：紫色系（#7C3AED, #6D28D9）表示权威
   - 强调色：金色/橙色（#F59E0B, #D97706）表示品质
   - **背景：透明**（不要添加背景矩形或背景色）
   - 使用线性渐变和径向渐变增强立体感

5. **技术要求**：
   - 使用SVG格式，包含完整的XML命名空间
   - 使用 `<defs>` 定义渐变和滤镜效果
   - 使用 `<linearGradient>` 和 `<radialGradient>` 创建渐变效果
   - 使用 `<filter>` 创建模糊、发光等特效
   - 使用 `<g>` 组织相关元素
   - 所有路径使用 `<path>` 或基本形状（`<rect>`, `<circle>`, `<ellipse>`）
   - 使用 `fill` 和 `stroke` 属性
   - 包含适当的 `opacity` 和 `transform` 属性

6. **设计特点**：
   - 等距投影（isometric）视角，3D立体感
   - 扁平化设计，简洁现代
   - 元素之间有层次感，使用阴影和渐变
   - 整体构图平衡，视觉焦点在中心
   - 适合浅色背景，在深色主题下也能良好显示

7. **动画兼容性**：
   - SVG代码应该能够配合CSS动画（上下浮动）
   - 避免使用内联动画，保持静态SVG
   - 确保所有元素都在viewBox范围内

8. **代码结构**：
   ```xml
   <svg
     enable-background="new 0 0 800 800"
     version="1.1"
     viewBox="0 0 800 800"
     xmlns="http://www.w3.org/2000/svg"
     xmlns:xlink="http://www.w3.org/1999/xlink"
   >
     <defs>
       <!-- 渐变定义 -->
       <!-- 滤镜定义 -->
     </defs>
     <!-- SVG内容 -->
   </svg>
````

9. **参考风格**：
   - 类似Undraw.co的插画风格
   - 等距投影3D效果
   - 现代扁平化设计
   - 专业、简洁、友好

**输出要求：**

- 只输出完整的SVG代码，不要包含任何解释文字
- SVG代码应该可以直接使用，无需修改
- 确保代码格式良好，缩进清晰
- 所有ID应该唯一
- 使用有意义的元素ID命名（如 law-book, scale, folder等）

```

## 📋 详细版本提示词（包含更多细节）

```

请生成一个法律行业管理系统的登录页面SVG插画，要求如下：

**视觉风格：**

- 现代扁平化设计
- 等距投影（isometric projection）3D风格
- 简洁、专业、友好
- 类似Undraw.co或DrawKit的插画风格

**画布设置：**

- viewBox="0 0 800 800"
- 适合横向展示（宽大于高）
- 所有元素在画布范围内

**核心元素（必须包含）：**

1. **法律书籍/法典**（中心位置）
   - 3-4本堆叠的书籍
   - 等距投影视角
   - 使用深蓝色渐变（#1E40AF 到 #3B82F6）
   - 添加阴影和高光效果

2. **天平**（正义象征）
   - 等距投影的天平
   - 位于书籍上方或旁边
   - 使用金色或深蓝色（#F59E0B 或 #1E40AF）
   - 平衡状态，象征公正

3. **文件夹/卷宗**
   - 2-3个文件夹或卷宗
   - 部分打开，显示文档
   - 使用浅蓝色和白色（#DBEAFE, #FFFFFF）
   - 堆叠或散落状态

4. **人物形象**（可选）
   - 简化的等距投影小人
   - 律师或法官形象
   - 站在平台上或与元素互动
   - 使用深色（#1F2937）

5. **数据图表**
   - 柱状图或折线图
   - 表示数据分析功能
   - 使用渐变色（蓝色到紫色）
   - 浮动在背景中

6. **云朵图标**
   - 表示云端存储
   - 使用浅蓝色（#E0F2FE）
   - 半透明效果

7. **发光平台/底座**
   - 圆形或方形的发光底座
   - 使用径向渐变
   - 蓝色或紫色光晕效果

**配色方案：**

- 主色：深蓝色 #1E40AF, #2563EB, #3B82F6
- 辅助色：紫色 #7C3AED, #6D28D9
- 强调色：金色 #F59E0B, #D97706
- 背景：浅蓝色渐变 #EFF6FF → #DBEAFE → #BFDBFE
- 文字/线条：深灰色 #1F2937, #374151
- 高光：白色 #FFFFFF
- 阴影：半透明黑色 rgba(0,0,0,0.1-0.3)

**技术实现：**

1. **渐变定义**（在 `<defs>` 中）：
   - 线性渐变：用于书籍、文件夹的立体感
   - 径向渐变：用于发光效果、圆形元素
   - 至少定义5-8个不同的渐变

2. **滤镜效果**（在 `<defs>` 中）：
   - 高斯模糊（feGaussianBlur）：用于阴影和光晕
   - 发光效果：用于重要元素
   - 至少定义2-3个滤镜

3. **元素组织**：
   - 使用 `<g>` 标签组织相关元素
   - 背景元素 → 主要元素 → 前景元素
   - 每个主要元素使用独立的 `<g>` 组

4. **路径和形状**：
   - 使用 `<path>` 绘制复杂形状
   - 使用基本形状（rect, circle, ellipse）绘制简单元素
   - 所有路径使用相对坐标（小写字母）

5. **变换和效果**：
   - 使用 `transform` 进行旋转、缩放、平移
   - 使用 `opacity` 创建层次感
   - 使用 `fill-opacity` 和 `stroke-opacity`

**代码结构示例：**

```xml
<svg enable-background="new 0 0 800 800" version="1.1" viewBox="0 0 800 800"
     xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink">
  <defs>
    <!-- 渐变定义 -->
    <linearGradient id="bookGradient" x1="0%" y1="0%" x2="100%" y2="100%">
      <stop offset="0%" style="stop-color:#1E40AF;stop-opacity:1" />
      <stop offset="100%" style="stop-color:#3B82F6;stop-opacity:1" />
    </linearGradient>
    <!-- 更多渐变... -->

    <!-- 滤镜定义 -->
    <filter id="glow">
      <feGaussianBlur stdDeviation="3" result="coloredBlur"/>
      <feMerge>
        <feMergeNode in="coloredBlur"/>
        <feMergeNode in="SourceGraphic"/>
      </feMerge>
    </filter>
    <!-- 更多滤镜... -->
  </defs>

  <!-- 背景渐变 -->
  <rect width="800" height="800" fill="url(#backgroundGradient)"/>

  <!-- 主要元素组 -->
  <g id="mainElements">
    <!-- 法律书籍 -->
    <g id="lawBooks">
      <!-- 书籍路径 -->
    </g>

    <!-- 天平 -->
    <g id="scale">
      <!-- 天平路径 -->
    </g>

    <!-- 其他元素... -->
  </g>
</svg>
```

**输出格式：**

- 只输出完整的SVG XML代码
- 代码应该格式良好，有适当的缩进
- 所有ID唯一且有意义
- 注释使用 <!-- --> 格式
- 确保代码可以直接复制使用

**质量要求：**

- 代码行数：500-2000行（根据复杂度）
- 文件大小：尽量控制在50KB以内
- 所有路径闭合
- 颜色值使用十六进制或RGB
- 避免使用外部资源引用

