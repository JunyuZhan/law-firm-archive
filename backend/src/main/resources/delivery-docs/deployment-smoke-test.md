# 部署后冒烟测试流程

用于律师事务所电子档案系统首次部署、升级后复测或客户交付验收。

核心目标：

- 入库可用
- 保存可用
- 借阅可用

## 1. 基础健康检查

- `GET /api/actuator/health` 返回 `UP`
- 前端首页返回 `200`
- 对外域名可正常打开

## 2. 登录与鉴权

至少验证：

- 系统管理员可登录
- 档案员可登录
- 普通用户可登录
- `GET /api/auth/me` 返回当前用户信息

## 3. 入库验证

建议创建 1 条测试档案，至少包含：

- 标题
- 来源编号
- 案号
- 案件名称
- 委托人
- 承办律师
- 保管期限

验收点：

- 接口或手工建档返回成功
- 后台列表可查到档案
- 档案详情字段完整

## 4. 保存验证

由档案员执行：

1. 打开刚创建的档案详情
2. 上传 1 个电子文件
3. 执行正式归档
4. 再次查看档案状态

验收点：

- 电子文件已显示
- 文件数量大于 0
- 档案状态为已归档或 `STORED`
- 未正式归档档案不会被错误允许借阅

## 5. 借阅验证

由普通用户和档案员配合执行：

1. 普通用户提交借阅申请
2. 档案员审批通过
3. 档案员借出或生成电子借阅链接
4. 访问借阅链接进行预览或下载验证

验收点：

- 借阅申请成功
- 审批状态流转正确
- 电子借阅访问有效
- 预览或下载链路可用

## 6. 通过标准

以下条件同时满足视为通过：

- 全部核心容器健康
- 登录正常
- 入库正常
- 保存正常
- 借阅正常
- 公开访问正常
