## 一、项目概述

### 1.1 项目背景

中小型律所（20-50人）在日常运营中面临以下痛点：
- 客户信息分散，案源追踪困难
- 项目管理依赖Excel，协作效率低
- 财务核销手工操作，提成计算复杂易错
- 文书版本混乱，印章管理不规范
- 知识经验难以沉淀和复用

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **务实可行** | 单体起步，避免过度设计 |
| **业务驱动** | 覆盖八大核心领域，贴合律师工作习惯 |
| **分层清晰** | 依赖层→通用层→领域层→应用层→接口层 |
| **渐进演进** | 预留拆分能力，按需扩展 |
| **成本可控** | 硬件3-5万，总投入15-25万 |

### 1.3 目标用户

| 角色 | 主要功能 |
|------|----------|
| 合伙人 | 全局数据、审批决策、业绩分析 |
| 主办律师 | 项目管理、文书处理、工时记录 |
| 协办律师 | 任务执行、文书协作、进度更新 |
| 财务人员 | 收款核销、提成计算、报表统计 |
| 行政人员 | 考勤管理、资产管理、后勤支持 |
| 实习律师 | 辅助工作、学习成长 |

| 层级 | 技术选型 | 版本 | 说明 |

| | Pinia | 2.0+ | 状态管理 |
| **后端** | Spring Boot | 3.2+ | 应用框架 |
| | MyBatis-Plus | 3.5+ | ORM框架 |
| | Sa-Token | 1.37+ | 认证授权 |
| | Flowable | 7.0+ | 工作流引擎 |

| **数据库** | PostgreSQL | 15+ | 主数据库 |
| | Redis | 7.0+ | 缓存 |
| | Elasticsearch | 8.11+ | 全文检索 |
| **存储** | MinIO | 最新 | 对象存储 |
| **OCR** | PaddleOCR | 2.7+ | 文字识别 |
| **部署** | Docker + Compose | 24+ | 容器化部署 |
| **代理** | Nginx | 1.24+ | 反向代理 |

### 3.3 为什么选择单体架构

| 对比项 | 微服务架构 | 单体+模块化 |
|--------|-----------|-------------|
| 适用规模 | 100+人，高并发 | 20-50人，中低并发 |
| 开发复杂度 | 高 | 低 |
| 运维成本 | 高（10+服务） | 低（1个应用） |
| 硬件需求 | 64G+内存 | 16-32G内存 |
| 部署时间 | 长 | 短 |
| 调试难度 | 高（分布式追踪） | 低（单进程） |
| 事务处理 | 复杂（分布式事务） | 简单（本地事务） |

**结论**：20-50人律所，单体架构完全够用，且开发运维成本低。

### 4.2 层级依赖规则

```
┌─────────────────────────────────────────────────────────────┐
│                    interfaces (接口层)                       │
│                         ↓ 调用                              │
├─────────────────────────────────────────────────────────────┤
│                    application (应用层)                      │
│                         ↓ 编排                              │
├─────────────────────────────────────────────────────────────┤
│                      domain (领域层)                         │
│                         ↓ 定义接口                          │
├─────────────────────────────────────────────────────────────┤
│                  infrastructure (基础设施层)                 │
│                         ↑ 实现接口                          │
├─────────────────────────────────────────────────────────────┤
│                      common (通用层)                         │
│                    ↑ 被所有层依赖                            │
└─────────────────────────────────────────────────────────────┘
```

**依赖原则**：
- 上层可以依赖下层，下层不能依赖上层
- 领域层定义仓储接口，基础设施层实现
- 通用层被所有层依赖，但不依赖任何业务层
- 领域层是核心，不依赖任何框架

### 4.3 各层职责说明

| 层级 | 职责 | 示例 |
|------|------|------|
| **interfaces** | 处理HTTP请求，参数校验，调用应用服务 | Controller、DTO校验 |
| **application** | 编排领域对象，处理事务，跨域协调 | 收案流程、核销流程 |
| **domain** | 核心业务逻辑，业务规则，领域事件 | 提成计算、利冲检查 |
| **infrastructure** | 技术实现，数据访问，外部服务调用 | MyBatis、Redis、MinIO |
| **common** | 通用工具，异常定义，常量枚举 | 工具类、统一响应 |

```java
// domain/matter/entity/Matter.java
@Entity
@Table(name = "matter")
public class Matter {
    
    private Long id;
    private String matterNo;           // 项目编号
    private String matterName;         // 项目名称
    private Long clientId;             // 客户ID
    private BusinessType businessType; // 业务类型
    private MatterStatus status;       // 项目状态
    private BigDecimal contractAmount; // 合同金额
    private Long mainLawyerId;         // 主办律师
    private LocalDate applyDate;       // 申请日期
    
    // 业务规则：提交审批
    public void submitForApproval() {
        // 规则1：只有草稿状态可以提交
        if (this.status != MatterStatus.DRAFT) {
            throw new BusinessException("只有草稿状态的项目可以提交审批");
        }
        // 规则2：合同金额必须填写
        if (this.contractAmount == null || this.contractAmount.compareTo(BigDecimal.ZERO) <= 0) {
            throw new BusinessException("合同金额必须大于0");
        }
        // 规则3：必须指定主办律师
        if (this.mainLawyerId == null) {
            throw new BusinessException("必须指定主办律师");
        }
        
        // 状态流转
        this.status = MatterStatus.PENDING_APPROVAL;
        
        // 发布领域事件
        DomainEventPublisher.publish(new MatterSubmittedEvent(this.id, this.matterNo));
    }
    
    // 业务规则：审批通过
    public void approve(Long approverId, String comment) {
        if (this.status != MatterStatus.PENDING_APPROVAL) {
            throw new BusinessException("当前状态不允许审批");
        }
        this.status = MatterStatus.IN_PROGRESS;
        DomainEventPublisher.publish(new MatterApprovedEvent(this.id, approverId, comment));
    }
    
    // 业务规则：结案
    public void close(String closeSummary) {
        if (this.status != MatterStatus.IN_PROGRESS) {
            throw new BusinessException("只有进行中的项目可以结案");
        }
        this.status = MatterStatus.CLOSED;
        DomainEventPublisher.publish(new MatterClosedEvent(this.id, closeSummary));
    }
}
```

```java
// domain/finance/service/CommissionCalculator.java
@DomainService
public class CommissionCalculator {
    
    /**
     * 计算提成
     * 业务规则：
     * 1. 净收入 = 到账金额 - 税费 - 管理费 - 办案成本
     * 2. 提成 = 净收入 × 分配比例 × 提成比例
     * 3. 提成比例根据金额阶梯计算
     */
    public CommissionResult calculate(Payment payment, Matter matter, 
                                       List<MatterParticipant> participants,
                                       CommissionRule rule) {
        
        // 1. 计算净收入
        BigDecimal grossAmount = payment.getAmount();
        BigDecimal taxAmount = grossAmount.multiply(rule.getTaxRate());
        BigDecimal managementFee = grossAmount.multiply(rule.getManagementFeeRate());
        BigDecimal costAmount = payment.getCostAmount() != null ? payment.getCostAmount() : BigDecimal.ZERO;
        BigDecimal netAmount = grossAmount.subtract(taxAmount).subtract(managementFee).subtract(costAmount);
        
        // 2. 计算每个参与人的提成
        List<CommissionDetail> details = new ArrayList<>();
        
        for (MatterParticipant participant : participants) {
            // 分配比例
            BigDecimal distributionRatio = participant.getDistributionRatio();
            
            // 该参与人的净收入份额
            BigDecimal participantNetAmount = netAmount.multiply(distributionRatio);
            
            // 根据阶梯计算提成比例
            BigDecimal commissionRate = calculateTieredRate(participantNetAmount, rule.getRateTiers());
            
            // 提成金额
            BigDecimal commissionAmount = participantNetAmount.multiply(commissionRate);
            
            CommissionDetail detail = CommissionDetail.builder()
                .lawyerId(participant.getLawyerId())
                .grossAmount(grossAmount.multiply(distributionRatio))
                .taxAmount(taxAmount.multiply(distributionRatio))
                .managementFeeAmount(managementFee.multiply(distributionRatio))
                .costAmount(costAmount.multiply(distributionRatio))
                .netAmount(participantNetAmount)
                .distributionRatio(distributionRatio)
                .commissionRate(commissionRate)
                .commissionAmount(commissionAmount)
                .build();
            
            details.add(detail);
        }
        
        return new CommissionResult(payment.getId(), matter.getId(), details);
    }
    
    /**
     * 阶梯提成比例计算
     */
    private BigDecimal calculateTieredRate(BigDecimal amount, List<RateTier> tiers) {
        for (RateTier tier : tiers) {
            if (amount.compareTo(tier.getMinAmount()) >= 0 
                && (tier.getMaxAmount() == null || amount.compareTo(tier.getMaxAmount()) < 0)) {
                return tier.getRate();
            }
        }
        return tiers.get(tiers.size() - 1).getRate(); // 默认最高档
    }
}
```

```java
// application/matter/MatterIntakeAppService.java
@Service
@Transactional
public class MatterIntakeAppService {
    
    private final MatterRepository matterRepository;
    private final ClientRepository clientRepository;
    private final ConflictCheckDomainService conflictCheckService;
    private final WorkflowService workflowService;
    private final IdGenerator idGenerator;
    
    /**
     * 创建项目并提交审批
     * 用例流程：
     * 1. 验证客户存在
     * 2. 验证利冲审查通过
     * 3. 创建项目
     * 4. 提交审批
     * 5. 启动审批流程
     */
    public MatterDTO createAndSubmit(CreateMatterCommand cmd) {
        // 1. 验证客户
        Client client = clientRepository.findById(cmd.getClientId())
            .orElseThrow(() -> new BusinessException("客户不存在"));
        
        // 2. 验证利冲审查
        conflictCheckService.ensurePassed(cmd.getClientId(), cmd.getOpposingParty());
        
        // 3. 创建项目
        Matter matter = Matter.builder()
            .matterNo(idGenerator.generateMatterNo())
            .matterName(cmd.getMatterName())
            .clientId(cmd.getClientId())
            .businessType(cmd.getBusinessType())
            .contractNo(cmd.getContractNo())
            .contractAmount(cmd.getContractAmount())
            .mainLawyerId(cmd.getMainLawyerId())
            .applyDate(LocalDate.now())
            .status(MatterStatus.DRAFT)
            .build();
        
        // 4. 提交审批（领域逻辑）
        matter.submitForApproval();
        
        // 5. 持久化
        matterRepository.save(matter);
        
        // 6. 启动工作流
        String processInstanceId = workflowService.startProcess(
            "matter_intake_approval",
            matter.getId().toString(),
            Map.of(
                "matterId", matter.getId(),
                "amount", matter.getContractAmount(),
                "applicantId", cmd.getApplicantId()
            )
        );
        
        // 7. 返回DTO
        return MatterAssembler.toDTO(matter, processInstanceId);
    }
    
    /**
     * 审批通过
     */
    public void approve(Long matterId, Long approverId, String comment) {
        Matter matter = matterRepository.findById(matterId)
            .orElseThrow(() -> new BusinessException("项目不存在"));
        
        // 领域逻辑
        matter.approve(approverId, comment);
        
        // 持久化
        matterRepository.save(matter);
        
        // 完成工作流任务
        workflowService.completeTask(matterId.toString(), "approve", comment);
    }
}
```

### 6.5 Flowable 工作流配置

```xml
<!-- processes/matter_intake_approval.bpmn20.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<definitions xmlns="http://www.omg.org/spec/BPMN/20100524/MODEL"
             xmlns:flowable="http://flowable.org/bpmn"
             targetNamespace="http://lawfirm.com/matter">

  <process id="matter_intake_approval" name="项目收案审批" isExecutable="true">
    
    <!-- 开始节点 -->
    <startEvent id="start" name="开始"/>
    
    <!-- 利冲审查 -->
    <userTask id="conflict_check" name="利益冲突审查"
              flowable:candidateGroups="conflict_checker">
      <extensionElements>
        <flowable:formProperty id="checkResult" name="审查结果" type="enum" required="true">
          <flowable:value id="PASS" name="通过"/>
          <flowable:value id="CONFLICT" name="存在冲突"/>
        </flowable:formProperty>
      </extensionElements>
    </userTask>
    
    <!-- 利冲审查网关 -->
    <exclusiveGateway id="conflict_gateway" name="利冲结果判断"/>
    
    <!-- 金额判断网关 -->
    <exclusiveGateway id="amount_gateway" name="金额判断"/>
    
    <!-- 部门主任审批 -->
    <userTask id="dept_approval" name="部门主任审批"
              flowable:assignee="${deptManagerId}">
      <extensionElements>
        <flowable:formProperty id="approved" name="审批结果" type="boolean" required="true"/>
        <flowable:formProperty id="comment" name="审批意见" type="string"/>
      </extensionElements>
    </userTask>
    
    <!-- 合伙人审批 -->
    <userTask id="partner_approval" name="合伙人审批"
              flowable:candidateGroups="partner">
      <extensionElements>
        <flowable:formProperty id="approved" name="审批结果" type="boolean" required="true"/>
        <flowable:formProperty id="comment" name="审批意见" type="string"/>
      </extensionElements>
    </userTask>
    
    <!-- 财务审核 -->
    <userTask id="finance_review" name="财务审核"
              flowable:candidateGroups="finance_manager">
      <extensionElements>
        <flowable:formProperty id="approved" name="审批结果" type="boolean" required="true"/>
        <flowable:formProperty id="comment" name="审批意见" type="string"/>
      </extensionElements>
    </userTask>
    
    <!-- 审批结果网关 -->
    <exclusiveGateway id="approval_gateway" name="审批结果判断"/>
    
    <!-- 审批通过处理 -->
    <serviceTask id="approve_handler" name="审批通过处理"
                 flowable:delegateExpression="${matterApproveHandler}"/>
    
    <!-- 审批驳回处理 -->
    <serviceTask id="reject_handler" name="审批驳回处理"
                 flowable:delegateExpression="${matterRejectHandler}"/>
    
    <!-- 结束节点 -->
    <endEvent id="end_approved" name="审批通过"/>
    <endEvent id="end_rejected" name="审批驳回"/>
    <endEvent id="end_conflict" name="存在冲突"/>
    
    <!-- 流程连线 -->
    <sequenceFlow id="flow1" sourceRef="start" targetRef="conflict_check"/>
    <sequenceFlow id="flow2" sourceRef="conflict_check" targetRef="conflict_gateway"/>
    
    <sequenceFlow id="flow3" sourceRef="conflict_gateway" targetRef="amount_gateway">
      <conditionExpression>${checkResult == 'PASS'}</conditionExpression>
    </sequenceFlow>
    <sequenceFlow id="flow4" sourceRef="conflict_gateway" targetRef="end_conflict">
      <conditionExpression>${checkResult == 'CONFLICT'}</conditionExpression>
    </sequenceFlow>
    
    <sequenceFlow id="flow5" sourceRef="amount_gateway" targetRef="dept_approval">
      <conditionExpression>${amount &lt; 50000}</conditionExpression>
    </sequenceFlow>
    <sequenceFlow id="flow6" sourceRef="amount_gateway" targetRef="partner_approval">
      <conditionExpression>${amount >= 50000}</conditionExpression>
    </sequenceFlow>
    
    <sequenceFlow id="flow7" sourceRef="dept_approval" targetRef="finance_review"/>
    <sequenceFlow id="flow8" sourceRef="partner_approval" targetRef="finance_review"/>
    
    <sequenceFlow id="flow9" sourceRef="finance_review" targetRef="approval_gateway"/>
    
    <sequenceFlow id="flow10" sourceRef="approval_gateway" targetRef="approve_handler">
      <conditionExpression>${approved == true}</conditionExpression>
    </sequenceFlow>
    <sequenceFlow id="flow11" sourceRef="approval_gateway" targetRef="reject_handler">
      <conditionExpression>${approved == false}</conditionExpression>
    </sequenceFlow>
    
    <sequenceFlow id="flow12" sourceRef="approve_handler" targetRef="end_approved"/>
    <sequenceFlow id="flow13" sourceRef="reject_handler" targetRef="end_rejected"/>
    
  </process>
```

```python
_ocr_engine = None

def get_ocr_engine():
    global _ocr_engine

class BankStatementParser:
    """银行回单解析器"""
    
    @staticmethod
    def parse_amount(text: str) -> Optional[float]:
        patterns = [
            r'金额[：:]\s*([￥¥]?\s*[\d,]+\.?\d*)',
            r'人民币[：:]\s*([\d,]+\.?\d*)\s*元',
            r'小写[：:]\s*([￥¥]?\s*[\d,]+\.?\d*)',
            r'[￥¥]\s*([\d,]+\.?\d*)',
        ]
        for pattern in patterns:
            match = re.search(pattern, text)
            if match:
                amount_str = match.group(1).replace(',', '').replace('¥', '').replace('￥', '').strip()
                try:
                    return float(amount_str)
                except:
                    continue
        return None
    
    @staticmethod
    def parse_date(text: str) -> Optional[str]:
        patterns = [
            r'(\d{4})年(\d{1,2})月(\d{1,2})日',
            r'(\d{4})-(\d{1,2})-(\d{1,2})',
            r'(\d{4})/(\d{1,2})/(\d{1,2})',
        ]
        for pattern in patterns:
            match = re.search(pattern, text)
            if match:
                year, month, day = match.groups()
                return f"{year}-{month.zfill(2)}-{day.zfill(2)}"
        return None
    
    @staticmethod
    def parse_payer(text: str) -> Optional[str]:
        patterns = [
            r'付款人[：:]\s*([\u4e00-\u9fa5a-zA-Z0-9]+)',
            r'付款账户[：:]\s*([\u4e00-\u9fa5a-zA-Z0-9]+)',
            r'汇款人[：:]\s*([\u4e00-\u9fa5a-zA-Z0-9]+)',
        ]
        for pattern in patterns:
            match = re.search(pattern, text)
            if match:
                return match.group(1).strip()
        return None


@app.post("/api/ocr/recognize")
async def recognize(file: UploadFile = File(...), doc_type: str = "auto"):
    """OCR 识别接口"""
    try:
        contents = await file.read()
        image = cv2.imdecode(np.frombuffer(contents, np.uint8), cv2.IMREAD_COLOR)
        
        if image is None:
            raise HTTPException(status_code=400, detail="无法读取图片")
        
        ocr_engine = get_ocr_engine()
        result = ocr_engine.ocr(image, cls=True)
        
        texts = []
        if result and result[0]:
            for line in result[0]:
                if line and len(line) >= 2:
                    _, (text, confidence) = line
                    texts.append({'text': text, 'confidence': float(confidence)})
        
        full_text = ' '.join([t['text'] for t in texts])
        
        # 结构化解析
        structured_data = {}
        if doc_type == 'bank_statement' or '银行' in full_text or '金额' in full_text:
            parser = BankStatementParser()
            structured_data = {
                'document_type': 'bank_statement',
                'amount': parser.parse_amount(full_text),
                'date': parser.parse_date(full_text),
                'payer': parser.parse_payer(full_text),
            }
        
        return JSONResponse(content={
            'success': True,
            'data': {
                'texts': texts,
                'structured_data': structured_data,
                'full_text': full_text
            }
        })
        
    except Exception as e:
        return JSONResponse(status_code=500, content={'success': False, 'message': str(e)})
```

```java
    public void confirmPayment(Long paymentId) {
        // 1. 确认收款
        Payment payment = paymentRepository.findById(paymentId).orElseThrow();
        payment.confirm();
        paymentRepository.save(payment);
        
        // 2. 异步计算提成（发送消息）
        rabbitTemplate.convertAndSend("commission.calculate",

    }
}

    }
}
```

### 14.7 权限校验实现

```java
// 权限注解
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
public @interface RequirePermission {
    String value();                    // 权限代码
    DataScope dataScope() default DataScope.DEFAULT;  // 数据范围
}

// 数据范围枚举
public enum DataScope {
    DEFAULT,      // 使用角色默认范围
    ALL,          // 全部数据
    DEPT,         // 本部门
    TEAM,         // 本团队
    SELF,         // 仅本人
    PARTICIPATED, // 参与的项目
    ASSIGNED      // 分配的数据
}

// 权限拦截器
@Aspect
@Component
public class PermissionAspect {
    
    @Autowired
    private PermissionService permissionService;
    
    @Around("@annotation(requirePermission)")
    public Object checkPermission(ProceedingJoinPoint point, RequirePermission requirePermission) throws Throwable {
        String permissionCode = requirePermission.value();
        Long userId = SecurityUtils.getCurrentUserId();
        
        // 检查功能权限
        if (!permissionService.hasPermission(userId, permissionCode)) {
            throw new AuthException("没有操作权限: " + permissionCode);
        }
        
        return point.proceed();
    }
}

// 数据权限处理器
@Component
public class DataScopeHandler implements DataPermissionHandler {
    
    @Override
    public Expression getSqlSegment(Expression where, String mappedStatementId) {
        Long userId = SecurityUtils.getCurrentUserId();
        DataScope dataScope = SecurityUtils.getCurrentDataScope();
        
        if (dataScope == DataScope.ALL) {
            return where;  // 不添加限制
        }
        
        // 根据数据范围添加SQL条件
        String scopeCondition = buildScopeCondition(userId, dataScope);
        if (scopeCondition != null) {
            Expression scopeExpression = CCJSqlParserUtil.parseCondExpression(scopeCondition);
            return where == null ? scopeExpression : new AndExpression(where, scopeExpression);
        }
        
        return where;
    }
    
    private String buildScopeCondition(Long userId, DataScope dataScope) {
        switch (dataScope) {
            case SELF:
                return "created_by = " + userId;
            case DEPT:
                Long deptId = userService.getDeptId(userId);
                return "department_id = " + deptId;
            case TEAM:
                Long teamId = userService.getTeamId(userId);
                return "team_id = " + teamId;
            case PARTICIPATED:
                return "id IN (SELECT matter_id FROM matter_participant WHERE user_id = " + userId + ")";
            case ASSIGNED:
                return "assigned_to = " + userId;
            default:
                return null;
        }
    }
}

// 使用示例
@RestController
@RequestMapping("/api/matters")
public class MatterController {
    
    @GetMapping
    @RequirePermission("matter:view")  // 自动应用数据范围过滤
    public Result<PageResult<MatterDTO>> list(MatterQueryRequest request) {
        return Result.success(matterService.queryPage(request));
    }
    
    @PostMapping
    @RequirePermission("matter:create")
    public Result<MatterDTO> create(@RequestBody CreateMatterRequest request) {
        return Result.success(matterService.create(request));
    }
    
    @PostMapping("/{id}/approve")
    @RequirePermission("matter:approve")
    public Result<Void> approve(@PathVariable Long id, @RequestBody ApproveRequest request) {
        matterService.approve(id, request);
        return Result.success();
    }
}
```

```java
@Service
public class ClientPortalService {
    
    public List<MatterDTO> getMyMatters(Long clientUserId) {
        // 获取客户关联的客户ID
        Long clientId = clientUserRepository.getClientId(clientUserId);
        
        // 只返回该客户的案件
        return matterRepository.findByClientId(clientId)
            .stream()
            .map(this::toClientViewDTO)  // 转换为客户视角的DTO，隐藏敏感信息

            .id(matter.getId())
            .matterNo(matter.getMatterNo())
            .matterName(matter.getMatterName())
            .status(matter.getStatus())
            .mainLawyerName(matter.getMainLawyer().getRealName())
            .currentStage(matter.getCurrentStage())
            // 不返回: 合同金额、提成信息、内部备注等
            .build();
    }
```

```typescript
interface BackendMenuItem {
  id: number;
  parentId: number;
  menuName: string;
  menuType: 'DIRECTORY' | 'MENU' | 'BUTTON';
  permissionCode?: string;
  path?: string;
  component?: string;
  icon?: string;
  sortOrder: number;
  visible: boolean;
  children?: BackendMenuItem[];
}

// 转换为 Vben 路由格式
function transformMenuToRoute(menus: BackendMenuItem[]): RouteRecordRaw[] {
  return menus
    .filter(menu => menu.menuType !== 'BUTTON' && menu.visible)
    .map(menu => ({
      path: menu.path || '',
      name: menu.permissionCode || menu.menuName,
      component: menu.component ? () => import(`@/views/${menu.component}.vue`) : LAYOUT,
      meta: {
        title: menu.menuName,
        icon: menu.icon,
        orderNo: menu.sortOrder,
      },
      children: menu.children ? transformMenuToRoute(menu.children) : undefined,
    }));
}
```

## 十六、数据迁移方案

### 16.1 迁移概述

大多数律所在上线新系统前，已有历史数据存储在 Excel、旧系统或纸质档案中。数据迁移是系统上线的关键环节，需要确保数据完整性、准确性和业务连续性。

### 16.2 迁移范围评估

#### 16.2.1 数据来源分析

| 数据来源 | 数据类型 | 迁移难度 | 优先级 |
|----------|----------|----------|--------|
| Excel 表格 | 客户、项目、收款 | 中 | P0 |
| 旧管理系统 | 全量业务数据 | 高 | P0 |
| 财务软件 | 收款、发票、提成 | 中 | P0 |
| 纸质档案 | 合同、文书 | 高 | P1 |
| 邮件系统 | 沟通记录 | 低 | P2 |

#### 16.2.2 数据迁移优先级

```
┌─────────────────────────────────────────────────────────────────┐
│                      数据迁移优先级                              │
├─────────────────────────────────────────────────────────────────┤
│  P0 必须迁移（上线前完成）                                       │
│  ├─ 客户基础信息                                                │
│  ├─ 进行中的项目                                                │
│  ├─ 未结清的收款记录                                            │
│  ├─ 用户账号和权限                                              │
│  └─ 基础字典数据                                                │
├─────────────────────────────────────────────────────────────────┤
│  P1 重要迁移（上线后1个月内完成）                                │
│  ├─ 历史已结案项目                                              │
│  ├─ 历史收款和提成记录                                          │
│  ├─ 客户联系人信息                                              │
│  └─ 文档模板                                                    │
├─────────────────────────────────────────────────────────────────┤
│  P2 可选迁移（按需进行）                                         │
│  ├─ 历史归档项目                                                │
│  ├─ 知识库文章                                                  │
│  └─ 历史操作日志                                                │
└─────────────────────────────────────────────────────────────────┘
```

### 16.3 迁移流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 数据收集 │───>│ 数据清洗 │───>│ 数据映射 │───>│ 试迁移  │───>│ 正式迁移 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
  收集所有      去重、补全      字段对应      测试环境      生产环境
  数据源        格式标准化      规则定义      验证数据      执行迁移
```

```python
    def import_clients(self, excel_path: str, sheet_name: str = '客户'):
        """导入客户数据到临时表"""
        df = pd.read_excel(excel_path, sheet_name=sheet_name)
        
        # 数据清洗
        df = df.dropna(subset=['客户名称'])  # 客户名称必填
        df = df.drop_duplicates(subset=['客户名称'])  # 去重
        
        # 字段映射
        records = []
        for _, row in df.iterrows():
            raw_data = row.to_dict()
            # 处理 NaN 值
            raw_data = {k: (None if pd.isna(v) else v) for k, v in raw_data.items()}
            
            record = {
                'source_id': str(row.get('编号', '')),
                'source_system': 'EXCEL',
                'raw_data': json.dumps(raw_data, ensure_ascii=False, default=str),
                'mapped_client_name': str(row['客户名称']).strip(),
                'mapped_client_type': self._map_client_type(row.get('客户类型', '')),
                'mapped_contact_person': str(row.get('联系人', '')).strip() or None,
                'mapped_contact_phone': str(row.get('电话', '')).strip() or None,
            }
            records.append(record)
        
        # 批量插入
        with self.conn.cursor() as cur:
            execute_values(cur, """
                INSERT INTO migration_client_temp 
                (source_id, source_system, raw_data, mapped_client_name, 
                 mapped_client_type, mapped_contact_person, mapped_contact_phone)
                VALUES %s
            """, [tuple(r.values()) for r in records])
        self.conn.commit()
        
        print(f"导入 {len(records)} 条客户数据到临时表")
        return len(records)
    
    def _map_client_type(self, value: str) -> str:
        """映射客户类型"""
        mapping = {
            '个人': 'PERSONAL',
            '企业': 'COMPANY',
            '公司': 'COMPANY',
            '政府': 'GOVERNMENT',
            '机关': 'GOVERNMENT',
        }
        return mapping.get(str(value).strip(), 'COMPANY')
    
    def import_matters(self, excel_path: str, sheet_name: str = '项目'):
        """导入项目数据到临时表"""
        df = pd.read_excel(excel_path, sheet_name=sheet_name)
        df = df.dropna(subset=['项目名称', '客户名称'])
        
        records = []
        for _, row in df.iterrows():
            raw_data = row.to_dict()
            raw_data = {k: (None if pd.isna(v) else v) for k, v in raw_data.items()}
            
            record = {
                'source_id': str(row.get('编号', '')),
                'source_system': 'EXCEL',
                'raw_data': json.dumps(raw_data, ensure_ascii=False, default=str),
                'mapped_matter_name': str(row['项目名称']).strip(),
                'mapped_client_name': str(row['客户名称']).strip(),
                'mapped_main_lawyer_name': str(row.get('主办律师', '')).strip() or None,
                'mapped_business_type': str(row.get('业务类型', '')).strip() or None,
                'mapped_status': self._map_matter_status(row.get('状态', '')),
                'mapped_contract_amount': self._parse_amount(row.get('合同金额')),
            }
            records.append(record)
        
        with self.conn.cursor() as cur:
            execute_values(cur, """
                INSERT INTO migration_matter_temp 
                (source_id, source_system, raw_data, mapped_matter_name, 
                 mapped_client_name, mapped_main_lawyer_name, mapped_business_type,
                 mapped_status, mapped_contract_amount)
                VALUES %s
            """, [tuple(r.values()) for r in records])
        self.conn.commit()
        
        print(f"导入 {len(records)} 条项目数据到临时表")
        return len(records)
    
    def _map_matter_status(self, value: str) -> str:
        mapping = {
            '进行中': 'IN_PROGRESS',
            '已结案': 'CLOSED',
            '已归档': 'ARCHIVED',
        }
        return mapping.get(str(value).strip(), 'IN_PROGRESS')
    
    def _parse_amount(self, value) -> float:
        if pd.isna(value):
            return None
        try:
            return float(str(value).replace(',', '').replace('￥', '').replace('元', ''))
        except:
            return None
```

#### 16.5.2 数据迁移执行脚本

```sql
-- migration/execute_migration.sql

-- 1. 迁移用户数据（需要先手动准备用户映射）
-- 假设用户已通过其他方式导入

-- 2. 迁移客户数据
INSERT INTO crm_client (
    client_no, client_name, client_type, 
    source_channel, cooperation_status,
    created_at, updated_at
)
SELECT 
    'C' || LPAD(nextval('client_no_seq')::text, 6, '0'),
    mapped_client_name,
    mapped_client_type,
    'MIGRATION',
    'ACTIVE',
    CURRENT_TIMESTAMP,
    CURRENT_TIMESTAMP
FROM migration_client_temp
WHERE status = 'PENDING'
RETURNING id, mapped_client_name;

-- 更新迁移状态
UPDATE migration_client_temp t
SET status = 'MIGRATED',
    migrated_id = c.id,
    migrated_at = CURRENT_TIMESTAMP
FROM crm_client c
WHERE t.mapped_client_name = c.client_name
  AND t.status = 'PENDING';

-- 3. 创建联系人记录
INSERT INTO crm_contact (
    client_id, contact_name, mobile_phone, is_primary
)
SELECT 
    t.migrated_id,
    t.mapped_contact_person,
    t.mapped_contact_phone,
    TRUE
FROM migration_client_temp t
WHERE t.status = 'MIGRATED'
  AND t.mapped_contact_person IS NOT NULL;

-- 4. 关联项目的客户ID
UPDATE migration_matter_temp m
SET linked_client_id = c.id
FROM crm_client c
WHERE m.mapped_client_name = c.client_name;

-- 5. 关联项目的主办律师ID
UPDATE migration_matter_temp m
SET linked_main_lawyer_id = u.id
FROM sys_user u
WHERE m.mapped_main_lawyer_name = u.real_name;

-- 6. 迁移项目数据
INSERT INTO matter (
    matter_no, matter_name, client_id, main_lawyer_id,
    business_category, business_type, status,
    created_at, updated_at
)
SELECT 
    'M' || LPAD(nextval('matter_no_seq')::text, 8, '0'),
    mapped_matter_name,
    linked_client_id,
    COALESCE(linked_main_lawyer_id, 1),  -- 默认管理员
    'NON_LITIGATION',
    COALESCE(mapped_business_type, '常年法律顾问'),
    mapped_status,
    CURRENT_TIMESTAMP,
    CURRENT_TIMESTAMP
FROM migration_matter_temp
WHERE status = 'PENDING'
  AND linked_client_id IS NOT NULL;

-- 更新迁移状态
UPDATE migration_matter_temp t
SET status = 'MIGRATED',
    migrated_id = m.id,
    migrated_at = CURRENT_TIMESTAMP
FROM matter m
WHERE t.mapped_matter_name = m.matter_name
  AND t.status = 'PENDING';

-- 7. 记录迁移失败的数据
UPDATE migration_client_temp
SET status = 'FAILED',
    error_message = '客户名称重复或数据不完整'
WHERE status = 'PENDING';

UPDATE migration_matter_temp
SET status = 'FAILED',
    error_message = CASE 
        WHEN linked_client_id IS NULL THEN '未找到对应客户'
        ELSE '数据不完整'
    END
WHERE status = 'PENDING';
```

### 16.7 迁移回滚

```sql
-- migration/rollback_migration.sql

-- 如果迁移出现问题，可以回滚
BEGIN;

-- 删除迁移的项目
DELETE FROM matter 
WHERE id IN (SELECT migrated_id FROM migration_matter_temp WHERE status = 'MIGRATED');

-- 删除迁移的联系人
DELETE FROM crm_contact 
WHERE client_id IN (SELECT migrated_id FROM migration_client_temp WHERE status = 'MIGRATED');

-- 删除迁移的客户
DELETE FROM crm_client 
WHERE id IN (SELECT migrated_id FROM migration_client_temp WHERE status = 'MIGRATED');

-- 重置迁移状态
UPDATE migration_client_temp SET status = 'PENDING', migrated_id = NULL, migrated_at = NULL;
UPDATE migration_matter_temp SET status = 'PENDING', migrated_id = NULL, migrated_at = NULL;

COMMIT;
```

### 16.8 迁移注意事项

| 注意事项 | 说明 |
|----------|------|
| **数据备份** | 迁移前必须备份原始数据和目标数据库 |
| **分批迁移** | 大量数据分批次迁移，每批1000-5000条 |
| **业务暂停** | 正式迁移期间暂停业务操作，避免数据冲突 |
| **双跑验证** | 新旧系统并行运行1-2周，对比数据一致性 |
| **用户培训** | 迁移完成后进行用户培训，确保能正确使用 |
| **应急预案** | 准备回滚方案，出现严重问题可快速恢复 |
