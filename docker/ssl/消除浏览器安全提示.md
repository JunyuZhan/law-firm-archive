# 消除浏览器安全提示指南

## 📋 问题说明

访问 `https://192.168.50.10` 时，浏览器会提示"您的连接不是私密连接"或"证书不受信任"，这是因为使用的是自签名证书。

## ✅ 解决方案

### 方案一：导入 CA 证书到浏览器（推荐，最简单）

#### Chrome/Edge 浏览器

1. **下载 CA 证书**
   ```bash
   # 在服务器上执行
   scp root@192.168.50.10:/opt/law-firm/docker/ssl/ca.crt ~/Downloads/
   ```

2. **导入证书**
   - Windows:
     - 打开 Chrome/Edge 设置 → 隐私和安全 → 安全 → 管理证书
     - 切换到"受信任的根证书颁发机构"标签
     - 点击"导入"，选择 `ca.crt` 文件
     - 勾选"将所有证书都放入下列存储" → "受信任的根证书颁发机构"
   
   - macOS:
     - 双击 `ca.crt` 文件
     - 打开"钥匙串访问"
     - 找到导入的证书，双击打开
     - 展开"信任"，将"使用此证书时"设置为"始终信任"
   
   - Linux:
     ```bash
     # 复制证书到系统信任存储
     sudo cp ca.crt /usr/local/share/ca-certificates/law-firm-ca.crt
     sudo update-ca-certificates
     
     # Chrome/Chromium 需要重启浏览器
     ```

3. **重启浏览器**，然后访问 `https://192.168.50.10`

#### Firefox 浏览器

1. **下载 CA 证书**（同上）

2. **导入证书**
   - 打开 Firefox → 设置 → 隐私与安全 → 证书 → 查看证书
   - 切换到"证书颁发机构"标签
   - 点击"导入"
   - 选择 `ca.crt` 文件
   - 勾选"信任此 CA 以标识网站"

3. **重启浏览器**，然后访问 `https://192.168.50.10`

### 方案二：导入 CA 证书到操作系统（推荐，全局生效）

#### Windows

1. **下载 CA 证书**
   ```bash
   scp root@192.168.50.10:/opt/law-firm/docker/ssl/ca.crt ~/Downloads/
   ```

2. **导入到系统**
   - 双击 `ca.crt` 文件
   - 点击"安装证书"
   - 选择"本地计算机" → 下一步
   - 选择"将所有证书都放入下列存储" → 浏览 → 选择"受信任的根证书颁发机构"
   - 完成

3. **重启浏览器**，访问 `https://192.168.50.10`

#### macOS

1. **下载 CA 证书**
   ```bash
   scp root@192.168.50.10:/opt/law-firm/docker/ssl/ca.crt ~/Downloads/
   ```

2. **导入到系统**
   ```bash
   # 双击 ca.crt 文件，会自动打开"钥匙串访问"
   # 或者使用命令行：
   sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ~/Downloads/ca.crt
   ```

3. **设置信任**
   - 打开"钥匙串访问"（应用程序 → 实用工具）
   - 在"系统"钥匙串中找到导入的证书
   - 双击证书，展开"信任"
   - 将"使用此证书时"设置为"始终信任"

4. **重启浏览器**，访问 `https://192.168.50.10`

#### Linux (Ubuntu/Debian)

```bash
# 1. 下载证书
scp root@192.168.50.10:/opt/law-firm/docker/ssl/ca.crt ~/

# 2. 复制到系统信任存储
sudo cp ~/ca.crt /usr/local/share/ca-certificates/law-firm-ca.crt

# 3. 更新证书存储
sudo update-ca-certificates

# 4. 重启浏览器
```

### 方案三：使用 Let's Encrypt 免费证书（生产环境推荐）

如果服务器有公网IP和域名，可以使用 Let's Encrypt 获取受信任的免费证书：

```bash
# SSH 到服务器
ssh root@192.168.50.10

# 安装 certbot
apt update
apt install certbot -y

# 获取证书（需要域名已解析到服务器）
certbot certonly --standalone -d yourdomain.com

# 复制证书到项目目录
cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem /opt/law-firm/docker/ssl/fullchain.pem
cp /etc/letsencrypt/live/yourdomain.com/privkey.pem /opt/law-firm/docker/ssl/privkey.pem

# 设置权限
chmod 644 /opt/law-firm/docker/ssl/fullchain.pem
chmod 600 /opt/law-firm/docker/ssl/privkey.pem

# 重启前端服务
cd /opt/law-firm/docker
docker compose -f docker-compose.prod.yml restart frontend
```

**优点：**
- ✅ 浏览器完全信任，无安全提示
- ✅ 免费
- ✅ 自动续期（90天有效期）

**缺点：**
- ❌ 需要公网IP和域名
- ❌ 需要域名解析到服务器

## 📝 快速操作脚本

### 下载 CA 证书脚本

创建 `download-ca-cert.sh`：

```bash
#!/bin/bash
SERVER_IP="192.168.50.10"
SERVER_USER="root"
CERT_PATH="/opt/law-firm/docker/ssl/ca.crt"

echo "正在下载 CA 证书..."
scp ${SERVER_USER}@${SERVER_IP}:${CERT_PATH} ~/Downloads/law-firm-ca.crt

if [ $? -eq 0 ]; then
    echo "✅ 证书已下载到: ~/Downloads/law-firm-ca.crt"
    echo ""
    echo "下一步："
    echo "1. macOS: 双击证书文件，在钥匙串访问中设置为'始终信任'"
    echo "2. Windows: 双击证书文件，导入到'受信任的根证书颁发机构'"
    echo "3. Linux: sudo cp ~/Downloads/law-firm-ca.crt /usr/local/share/ca-certificates/ && sudo update-ca-certificates"
else
    echo "❌ 下载失败"
fi
```

## ⚠️ 注意事项

1. **安全提醒**
   - CA 证书是根证书，导入后可以签发任何域名的证书
   - 只在信任的环境中导入（如内网环境）
   - 不要将 CA 私钥泄露

2. **证书有效期**
   - 当前服务器证书有效期：365天
   - 到期前需要重新生成：运行 `generate-server-cert.sh` 脚本

3. **多台电脑访问**
   - 每台需要访问的电脑都需要导入 CA 证书
   - 或者使用 Let's Encrypt 证书（浏览器自动信任）

## 🆘 常见问题

### Q: 导入证书后还是提示不安全？

**A:** 
1. 确认导入的是 CA 证书（`ca.crt`），不是服务器证书
2. 确认导入到了"受信任的根证书颁发机构"
3. 清除浏览器缓存和 SSL 状态，重启浏览器
4. 检查证书链是否完整

### Q: 如何查看证书信息？

**A:**
```bash
# 查看服务器证书
openssl x509 -in /opt/law-firm/docker/ssl/server.crt -text -noout

# 查看 CA 证书
openssl x509 -in /opt/law-firm/docker/ssl/ca.crt -text -noout
```

### Q: 证书过期了怎么办？

**A:**
```bash
# 重新生成服务器证书
cd /opt/law-firm
./scripts/generate-server-cert.sh 192.168.50.10

# 上传并替换
scp docker/ssl/server.crt docker/ssl/server.key docker/ssl/fullchain.pem root@192.168.50.10:/opt/law-firm/docker/ssl/

# 在服务器上替换并重启
ssh root@192.168.50.10
cd /opt/law-firm/docker/ssl
cat server.crt ca.crt > fullchain.pem
cp server.key privkey.pem
cd ..
docker compose -f docker-compose.prod.yml restart frontend
```
