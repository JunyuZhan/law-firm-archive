# SSL 证书简单说明

## 🎯 一句话说明

**把SSL证书的两个文件放到 `docker/ssl/` 文件夹里就可以了！**

---

## 📂 具体位置

```
项目根目录/
└── docker/
    └── ssl/              ← 证书就放这里！
        ├── fullchain.pem  ← 证书文件（你从证书商那里下载的）
        └── privkey.pem    ← 私钥文件（你从证书商那里下载的）
```

---

## 📋 你需要做什么？

### 步骤1：准备证书文件

你有两个证书文件：
- **证书文件**：可能是 `fullchain.pem`、`cert.pem`、`certificate.crt` 等名字
- **私钥文件**：可能是 `privkey.pem`、`key.pem`、`private.key` 等名字

### 步骤2：放到指定位置

把这两个文件复制到 `docker/ssl/` 目录：

```bash
# 进入项目目录
cd /Users/apple/Documents/Project/law-firm

# 复制证书文件（根据你的实际文件名修改）
cp /你的证书路径/fullchain.pem docker/ssl/fullchain.pem
cp /你的证书路径/privkey.pem docker/ssl/privkey.pem
```

### 步骤3：设置文件权限

```bash
chmod 644 docker/ssl/fullchain.pem
chmod 600 docker/ssl/privkey.pem
```

## ❓ 常见问题

### Q1: 我没有证书，怎么办？

**选项A：使用 Let's Encrypt（免费）**
```bash
# 安装 certbot
sudo apt-get install certbot

# 获取证书（需要你的域名已经解析到服务器）
sudo certbot certonly --standalone -d 你的域名.com

# 证书会自动生成在 /etc/letsencrypt/live/你的域名.com/
# 然后复制到项目目录
sudo cp /etc/letsencrypt/live/你的域名.com/fullchain.pem docker/ssl/fullchain.pem
sudo cp /etc/letsencrypt/live/你的域名.com/privkey.pem docker/ssl/privkey.pem
```

**选项B：从云服务商购买**
- 阿里云、腾讯云、AWS等都有SSL证书服务
- 购买后下载证书文件，放到 `docker/ssl/` 目录

**选项C：测试用自签名证书（仅开发测试）**
```bash
cd docker/ssl
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout privkey.pem \
  -out fullchain.pem \
  -subj "/CN=localhost"
```

### Q2: 证书文件名不一样怎么办？

没关系！只要：
1. 把证书文件重命名为 `fullchain.pem`
2. 把私钥文件重命名为 `privkey.pem`
3. 放到 `docker/ssl/` 目录

### Q3: 放好证书后还要做什么？

放好证书后，还需要：
1. 修改 `docker-compose.prod.yml` 文件（添加证书挂载和443端口）
2. 使用支持HTTPS的nginx配置

详细步骤看 `README.md` 文件。
