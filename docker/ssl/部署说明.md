# SSL 证书部署说明

## ⚠️ 重要提示

**SSL 证书文件不会通过 Git 同步！**

因为证书文件是敏感信息，已经在 `.gitignore` 中被排除，所以：

- ✅ **代码拉取后**：证书文件不会自动下载
- ✅ **必须单独上传**：每次部署都需要手动上传证书
- ✅ **安全考虑**：这是正确的做法，证书不应该提交到版本控制

---

## 📋 部署流程

### 1. 拉取代码（证书不会下载）

```bash
# 在服务器上
cd /opt/law-firm
git pull  # 或 git clone
```

**结果：** `docker/ssl/` 目录只有 `README.md`，没有证书文件。

### 2. 上传证书（必须手动操作）

**方式一：使用上传脚本（推荐）**

```bash
# 在本地项目目录执行
./scripts/upload-ssl-certs.sh <服务器IP> [用户名]

# 例如：
./scripts/upload-ssl-certs.sh 192.168.1.100 root
```

**方式二：手动上传**

```bash
# 在本地执行
scp docker/ssl/ca.crt root@你的服务器IP:/opt/law-firm/docker/ssl/
scp docker/ssl/ca.key root@你的服务器IP:/opt/law-firm/docker/ssl/

# 然后在服务器上配置
ssh root@你的服务器IP
cd /opt/law-firm/docker/ssl
ln -sf ca.crt fullchain.pem
ln -sf ca.key privkey.pem
chmod 644 fullchain.pem
chmod 600 privkey.pem
```

**方式三：在服务器上直接获取（Let's Encrypt）**

```bash
# 在服务器上执行
apt install certbot -y
certbot certonly --standalone -d 你的域名.com

mkdir -p /opt/law-firm/docker/ssl
cp /etc/letsencrypt/live/你的域名.com/fullchain.pem /opt/law-firm/docker/ssl/
cp /etc/letsencrypt/live/你的域名.com/privkey.pem /opt/law-firm/docker/ssl/
```

### 3. 验证证书

```bash
# 在服务器上检查
ls -la /opt/law-firm/docker/ssl/

# 应该看到：
# - fullchain.pem 或 ca.crt
# - privkey.pem 或 ca.key
```

## 🔄 更新代码时的流程

每次更新代码后：

```bash
# 1. 拉取最新代码
cd /opt/law-firm
git pull

# 2. 检查证书是否存在（如果不存在，需要重新上传）
ls -la docker/ssl/*.pem docker/ssl/*.crt 2>/dev/null || echo "需要上传证书！"

# 3. 如果证书不存在，上传证书
# 在本地执行：
./scripts/upload-ssl-certs.sh <服务器IP>

# 4. 重新部署
cd docker
docker compose -f docker-compose.prod.yml up -d --build
```

## 💡 最佳实践

### 1. 使用自动化脚本

将证书上传集成到部署脚本中：

```bash
# 在 deploy-to-server.sh 中添加证书上传步骤
./scripts/upload-ssl-certs.sh $SERVER_IP
```

### 2. 使用 Let's Encrypt（生产环境推荐）

- 证书自动续期
- 浏览器信任
- 免费使用

### 3. 证书备份

定期备份证书文件到安全位置：

```bash
# 备份证书
tar -czf ssl-backup-$(date +%Y%m%d).tar.gz docker/ssl/*.pem docker/ssl/*.crt
```

## ❓ 常见问题

### Q: 为什么证书不在 Git 仓库中？

**A:** 这是安全最佳实践：
- 证书是敏感信息，不应该公开
- 避免证书泄露风险
- 符合安全规范

### Q: 每次都要上传证书吗？

**A:** 是的，但可以：
- 使用自动化脚本简化流程
- 在服务器上使用 Let's Encrypt（自动续期）
- 将证书存储在安全的配置管理系统中

### Q: 证书文件丢失了怎么办？

**A:** 
- 如果有备份，恢复备份
- 如果是 Let's Encrypt，重新获取：`certbot certonly --standalone -d 你的域名.com`
- 如果是商业证书，联系证书提供商重新下载
