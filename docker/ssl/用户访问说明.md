# 用户访问 HTTPS 网站说明

## 📋 问题说明

访问 `https://192.168.50.10` 时，浏览器会提示"您的连接不是私密连接"或"证书不受信任"。

这是因为使用的是自签名证书，浏览器默认不信任。

---

## ✅ 解决方案

### 方案一：使用 Let's Encrypt 证书（推荐，如果有域名）

如果服务器有公网域名，可以使用 Let's Encrypt 获取受信任的免费证书：

**优点：**
- ✅ 浏览器完全信任，用户无需任何操作
- ✅ 免费
- ✅ 自动续期

**配置步骤：**
```bash
# SSH 到服务器
ssh root@192.168.50.10

# 安装 certbot
apt update && apt install certbot -y

# 获取证书（需要域名已解析到服务器）
certbot certonly --standalone -d yourdomain.com

# 复制证书到项目目录
cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem /opt/law-firm/docker/ssl/fullchain.pem
cp /etc/letsencrypt/live/yourdomain.com/privkey.pem /opt/law-firm/docker/ssl/privkey.pem

# 设置权限
chmod 644 /opt/law-firm/docker/ssl/fullchain.pem
chmod 600 /opt/law-firm/docker/ssl/privkey.pem

# 重启前端服务
cd /opt/law-firm/docker
docker compose -f docker-compose.prod.yml restart frontend
```

---

### 方案二：用户导入 CA 证书（内网环境）

如果服务器是内网环境，没有公网域名，需要用户在他们的电脑上导入 CA 证书。

#### 步骤1：下载 CA 证书

**方式一：从网站下载（推荐）**

在网站首页添加一个下载链接，指向 CA 证书：
```html
<a href="/ssl/ca.crt" download="law-firm-ca.crt">下载CA证书</a>
```

**方式二：提供下载地址**

告诉用户访问：`http://192.168.50.10/ssl/ca.crt` 下载证书

**方式三：通过邮件/文件共享发送**

将 `ca.crt` 文件发送给用户

#### 步骤2：用户导入证书

**Windows 用户：**
1. 双击下载的 `ca.crt` 文件
2. 点击"安装证书"
3. 选择"本地计算机" → 下一步
4. 选择"将所有证书都放入下列存储" → 浏览
5. 选择"受信任的根证书颁发机构" → 确定 → 完成
6. 重启浏览器

**macOS 用户：**
1. 双击下载的 `ca.crt` 文件
2. 在钥匙串访问中找到导入的证书
3. 双击证书，展开"信任"
4. 将"使用此证书时"设置为"始终信任"
5. 重启浏览器

**Linux 用户：**
```bash
sudo cp ca.crt /usr/local/share/ca-certificates/law-firm-ca.crt
sudo update-ca-certificates
# 重启浏览器
```

## 🎯 推荐方案

### 如果有公网域名
→ **使用 Let's Encrypt**（方案一）
- 用户无需任何操作
- 浏览器自动信任

### 如果是内网环境
→ **让用户导入 CA 证书**（方案二）
- 需要每个用户导入一次
- 或者提供一个简单的下载和导入指南

## 🔧 技术实现：在网站提供证书下载

在 Nginx 配置中添加一个路由，让用户可以直接下载 CA 证书：

```nginx
# 在 nginx-ssl.conf 中添加
location /ssl/ca.crt {
    alias /etc/nginx/ssl/ca.crt;
    default_type application/x-x509-ca-cert;
    add_header Content-Disposition 'attachment; filename="law-firm-ca.crt"';
}
```

这样用户访问 `https://192.168.50.10/ssl/ca.crt` 就可以直接下载证书。
