## 优势

| 特性 | 说明 |
|------|------|
| **成本低** | 只需 1 台应用服务器 + 1 台 NAS |
| **数据安全** | NAS RAID5 提供磁盘冗余（允许坏1块盘） |
| **运维简单** | NAS 有图形界面，易管理 |
| **扩展方便** | 需要更多空间时，直接扩展 NAS 磁盘 |

## 部署步骤

### 1. NAS 配置

#### 群晖 (Synology)

1. **创建存储池**（RAID5）
   - 控制面板 → 存储管理器 → 存储池
   - 选择至少 3 块硬盘，配置 RAID5

2. **安装 Docker**
   - 套件中心 → 搜索 Docker → 安装

3. **创建目录**
   ```
   /volume1/docker/law-firm/minio
   /volume1/docker/law-firm/backup
   ```

4. **部署 MinIO**
   ```bash
   # 上传 docker-compose.minio-nas.yml 到 NAS
   cd /volume1/docker/law-firm
   docker-compose -f docker-compose.minio-nas.yml up -d
   ```

#### 威联通 (QNAP)

1. 控制台 → 存储与快照总管 → 创建 RAID5 存储池
2. App Center → 安装 Container Station
3. 创建目录 `/share/Container/law-firm/minio`
4. 部署 MinIO（同上）

### 2. 应用服务器配置

1. **配置环境变量**
   ```bash
   cd /path/to/law-firm
   cp docker/env.example docker/.env
   vim docker/.env
   ```

   ```env
   # 数据库
   DB_PASSWORD=<强密码>
   REDIS_PASSWORD=<强密码>
   JWT_SECRET=<64位随机字符串>
   
   # MinIO（指向 NAS）
   NAS_IP=192.168.50.20
   MINIO_ACCESS_KEY=minioadmin
   MINIO_SECRET_KEY=<强密码>
   ```

2. **部署应用**
   ```bash
   docker-compose -f docker/docker-compose.nas.yml up -d
   ```

3. **配置数据库备份到 NAS**
   ```bash
   # 编辑备份脚本，设置远程路径
   vim scripts/db-auto-backup.sh
   
   # 修改配置
   REMOTE_BACKUP_ENABLED=true
   REMOTE_PATH="admin@192.168.50.20:/volume1/docker/law-firm/backup/"
   
   # 设置 SSH 免密登录（可选）
   ssh-keygen -t rsa
   ssh-copy-id admin@192.168.50.20
   
   # 启动定时备份
   ./scripts/db-auto-backup.sh --schedule
   ```

### 3. 验证部署

```bash
# 检查应用服务
docker ps

# 测试 MinIO 连接
curl http://192.168.50.20:9000/minio/health/live

# 访问系统
http://192.168.50.10
```

## NAS 推荐配置

### 小型律所（5-20人）

| 组件 | 配置 | 说明 |
|------|------|------|
| NAS | 群晖 DS220+ | 2盘位 |
| 硬盘 | 2 x 4TB | RAID1 镜像 |
| 预计容量 | 4TB 可用 | 足够存储 10 年文档 |

### 中型律所（20-50人）

| 组件 | 配置 | 说明 |
|------|------|------|
| NAS | 群晖 DS920+ | 4盘位 |
| 硬盘 | 4 x 4TB | RAID5 |
| 预计容量 | 12TB 可用 | 可扩展到 8 盘 |

### 大型律所（50人+）

| 组件 | 配置 | 说明 |
|------|------|------|
| NAS | 群晖 RS1221+ | 机架式 8盘 |
| 硬盘 | 8 x 8TB | RAID6 |
| 预计容量 | 48TB 可用 | 企业级可靠性 |

## 数据保护

### RAID 级别选择

| RAID | 最少磁盘 | 可用容量 | 允许故障 | 推荐 |
|------|----------|----------|----------|------|
| RAID1 | 2 | 50% | 1块 | 小型 |
| RAID5 | 3 | 67%~80% | 1块 | **中型推荐** |
| RAID6 | 4 | 50%~67% | 2块 | 大型/关键 |

### 备份策略

```
┌─────────────────────────────────────────────────────────┐
│                    数据保护层次                          │
├─────────────────────────────────────────────────────────┤
│  第1层: RAID5                                           │
│  └─ 磁盘故障保护（允许坏1块）                            │
├─────────────────────────────────────────────────────────┤
│  第2层: 定时快照                                        │
│  └─ NAS 内置快照功能（防误删除、勒索病毒）               │
├─────────────────────────────────────────────────────────┤
│  第3层: 异地备份                                        │
│  └─ 定期同步到云存储或另一台 NAS                        │
└─────────────────────────────────────────────────────────┘
```

### 群晖快照设置

1. 控制面板 → 共享文件夹 → 选择 minio 文件夹
2. 快照 → 设置
3. 启用定时快照：每日
4. 保留策略：保留 30 天

### 云备份（可选）

```bash
# 群晖 Hyper Backup 到阿里云 OSS
# 控制面板 → Hyper Backup → 创建
# 目标：S3 兼容存储
# 端点：oss-cn-hangzhou.aliyuncs.com
```

## 故障恢复

### 磁盘故障

1. NAS 会发出警报
2. 更换故障磁盘
3. RAID 自动重建（无需停机）

### 文件误删除

```bash
# 从 NAS 快照恢复
# 文件管理器 → 找到文件夹 → 右键 → 从快照恢复
```

### 整机故障

1. 新 NAS 上重建 RAID（如有）
2. 从云备份恢复数据
3. 重新部署 MinIO
