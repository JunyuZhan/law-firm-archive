# 律所管理系统与档案管理系统对接指南

## 概述

本文档描述了律所管理系统（主分支 `main`）与档案管理系统（分支 `feature/archive-management-system`）的对接方式。

## 系统架构

```
┌─────────────────────────────┐                 ┌─────────────────────────────┐
│     律所管理系统            │                 │     档案管理系统            │
│     (main分支)              │                 │     (archive分支)           │
│     端口: 8080              │                 │     端口: 8090              │
│                             │                 │                             │
│  ┌─────────────────────┐   │    HTTP POST    │   ┌─────────────────────┐   │
│  │ ArchiveAppService   │───┼────────────────►│   │ OpenApiController   │   │
│  │ 档案入库时推送       │   │                 │   │ /open/archive/receive│   │
│  └─────────────────────┘   │                 │   └─────────────────────┘   │
│                             │                 │                             │
│  ┌─────────────────────┐   │    HTTP POST    │   ┌─────────────────────┐   │
│  │ ArchiveCallback     │◄──┼────────────────┼───│ CallbackService     │   │
│  │ Controller          │   │    回调通知     │   │ 处理完成后回调       │   │
│  └─────────────────────┘   │                 │   └─────────────────────┘   │
└─────────────────────────────┘                 └─────────────────────────────┘
```

## 对接流程

### 1. 档案入库推送

当律所管理系统中的档案入库（`storeArchive`）时，会自动推送到档案管理系统：

```
律所系统创建档案 → 档案入库 → 异步推送到档案管理系统 → 档案系统处理 → 回调通知
```

### 2. 配置说明（后台动态配置）

配置在后台"**系统管理 > 外部集成**"中进行，无需修改配置文件。

#### 配置步骤：

1. 登录律所管理系统后台
2. 进入"系统管理 > 外部集成"
3. 找到"**档案管理系统**"（编码：ARCHIVE_SYSTEM）
4. 点击编辑，配置以下内容：

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| API地址 | 档案系统API基础地址 | `http://localhost:8090/api` |
| API Key | 身份验证密钥 | （在档案系统中生成） |
| API Secret | 签名验证密钥 | （在档案系统中生成） |

5. 点击"额外配置"，配置JSON格式的扩展参数：

```json
{
  "sourceCode": "LAW_FIRM_MAIN",
  "callbackUrl": "http://localhost:8080/api/archive/callback",
  "async": true,
  "connectTimeout": 5000,
  "readTimeout": 30000
}
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| sourceCode | 来源系统编码 | LAW_FIRM_MAIN |
| callbackUrl | 回调地址 | （本系统地址/api/archive/callback） |
| async | 是否异步处理 | true |
| connectTimeout | 连接超时(ms) | 5000 |
| readTimeout | 读取超时(ms) | 30000 |

6. 保存配置
7. 点击"测试连接"验证配置是否正确
8. 确认无误后点击"启用"

### 3. 在档案管理系统中注册来源系统

需要在档案管理系统中创建对应的外部来源配置：

```sql
INSERT INTO arc_external_source (
    source_code, source_name, source_type, 
    api_key, secret_key, enabled
) VALUES (
    'LAW_FIRM_MAIN', '律所管理系统', 'LAW_FIRM',
    'your-api-key', 'your-secret-key', true
);
```

## API接口

### 推送接口（档案系统提供）

**POST** `/api/open/archive/receive`

请求参数：
| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| sourceType | String | 是 | 来源类型：LAW_FIRM |
| sourceId | String | 是 | 来源系统档案ID |
| sourceNo | String | 否 | 来源系统档案编号 |
| callbackUrl | String | 否 | 回调URL |
| async | Boolean | 否 | 是否异步处理（默认true） |
| title | String | 是 | 档案题名 |
| archiveType | String | 是 | 档案类型 |
| retentionPeriod | String | 是 | 保管期限 |
| files | Array | 否 | 电子文件列表 |

### 回调接口（律所系统提供）

**POST** `/api/archive/callback`

回调参数：
| 字段 | 类型 | 说明 |
|-----|------|------|
| sourceId | String | 原始来源ID |
| sourceNo | String | 原始来源编号 |
| archiveId | Long | 档案系统生成的ID |
| archiveNo | String | 档案系统生成的编号 |
| status | String | 处理状态：COMPLETED/FAILED |
| message | String | 状态消息 |
| signature | String | 签名（用于验证） |
| timestamp | Long | 时间戳 |

## 数据映射

| 律所系统 | 档案系统 |
|---------|---------|

## 代码结构

律所管理系统中的对接代码：

```
backend/src/main/java/com/lawfirm/
├── infrastructure/external/archivesystem/
│   ├── ArchiveSystemClient.java          # HTTP客户端（从数据库读取配置）
│   ├── ArchiveSystemPushService.java     # 推送服务
│   └── dto/
│       ├── ArchivePushRequest.java       # 推送请求DTO
│       ├── ArchivePushResponse.java      # 推送响应DTO
│       └── ArchiveCallbackRequest.java   # 回调请求DTO
├── interfaces/rest/archive/
│   └── ArchiveCallbackController.java    # 回调接口
├── application/archive/service/
│   └── ArchiveAppService.java            # 档案入库时推送
├── application/system/service/
│   └── ExternalIntegrationAppService.java # 外部集成配置服务
└── domain/system/entity/
    └── ExternalIntegration.java          # 配置实体
```

数据库表：
- `sys_external_integration` - 外部系统集成配置表

## 测试验证

### 1. 启动档案管理系统

```bash
cd archive-system/docker
docker-compose up -d
```

### 2. 配置律所系统（后台操作）

1. 登录律所管理系统
2. 进入"系统管理 > 外部集成"
3. 配置"档案管理系统"的API地址和密钥
4. 点击"测试连接"
5. 启用该集成

### 3. 测试推送

在律所系统中创建并入库一个档案，观察是否成功推送到档案管理系统。

## 故障排查

### 推送失败

1. 检查档案管理系统是否正常运行
2. 检查网络连通性
3. 检查配置中的 `api-key` 和 `secret-key` 是否正确
4. 查看日志：`ArchiveSystemClient` 相关日志

### 回调失败

1. 检查 `callback-url` 配置是否正确
2. 检查签名验证是否通过
3. 查看日志：`ArchiveCallbackController` 相关日志
