# 文档、证据与档案管理补充方案

> 本文档是《智慧律所管理系统-完整开发方案》的补充，针对案件文档管理、证据管理和档案管理进行详细设计。

---

## 一、现有方案不足分析

| 问题 | 说明 |
|------|------|
| 文档分类不够细化 | 缺少诉讼材料、证据、法律文书等专业分类 |
| 证据管理缺失 | 没有证据目录、证据清单、质证意见等功能 |
| 档案管理简单 | 只有归档位置，缺少档案借阅、销毁等流程 |
| 文档与案件阶段脱节 | 文档没有关联到案件的具体阶段 |
| 缺少文档模板变量 | 模板生成时的变量替换机制不完善 |

### 2.2 增强的文档表结构

```sql
-- 文档分类表（树形结构）
CREATE TABLE doc_category (
    id BIGSERIAL PRIMARY KEY,
    category_code VARCHAR(50) NOT NULL UNIQUE,
    category_name VARCHAR(100) NOT NULL,
    parent_id BIGINT DEFAULT 0,
    category_type VARCHAR(50),           -- CASE(案件), CONTRACT(合同), ADMIN(行政), TEMPLATE(模板)
    applicable_business_types JSONB,     -- 适用的业务类型
    sort_order INT DEFAULT 0,
    icon VARCHAR(50),
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 增强的文档表
CREATE TABLE doc_document (
    id BIGSERIAL PRIMARY KEY,
    document_no VARCHAR(50) NOT NULL UNIQUE,
    
    -- 归属信息
    owner_type VARCHAR(20) NOT NULL,     -- MATTER(案件), CLIENT(客户), FIRM(律所)
    matter_id BIGINT REFERENCES matter(id),
    client_id BIGINT REFERENCES crm_client(id),
    
    -- 分类信息
    category_id BIGINT REFERENCES doc_category(id),
    document_type VARCHAR(50) NOT NULL,
    
    -- 文档信息
    document_name VARCHAR(200) NOT NULL,
    description TEXT,
    keywords VARCHAR(500),               -- 关键词，便于搜索
    
    -- 关联案件阶段
    matter_stage_id BIGINT REFERENCES matter_stage(id),
    
    -- 文件信息
    current_version INT DEFAULT 1,
    file_url VARCHAR(500),
    file_name VARCHAR(200),              -- 原始文件名
    file_size BIGINT,
    file_extension VARCHAR(20),
    mime_type VARCHAR(100),
    file_hash VARCHAR(64),               -- 文件哈希，用于去重和完整性校验
    
    -- 安全级别
    security_level VARCHAR(20) DEFAULT 'INTERNAL',  -- PUBLIC, INTERNAL, CONFIDENTIAL, SECRET
    
    -- 来源信息
    source_type VARCHAR(50),             -- UPLOAD(上传), SCAN(扫描), TEMPLATE(模板生成), RECEIVE(接收)
    source_from VARCHAR(200),            -- 来源说明
    received_date DATE,                  -- 收到日期（法院文书等）
    
    -- 模板信息
    template_id BIGINT REFERENCES doc_template(id),
    template_variables JSONB,            -- 模板变量值
    
    -- OCR信息
    ocr_status VARCHAR(20),              -- PENDING, PROCESSING, COMPLETED, FAILED
    ocr_content TEXT,                    -- OCR识别的文本内容
    
    -- 状态
    status VARCHAR(20) DEFAULT 'ACTIVE', -- ACTIVE, ARCHIVED, DELETED
    
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_by BIGINT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted BOOLEAN DEFAULT FALSE
);

-- 文档版本表
CREATE TABLE doc_version (
    id BIGSERIAL PRIMARY KEY,
    document_id BIGINT NOT NULL REFERENCES doc_document(id),
    version_number INT NOT NULL,
    
    -- 文件信息
    file_url VARCHAR(500) NOT NULL,
    file_name VARCHAR(200),
    file_size BIGINT,
    file_hash VARCHAR(64),
    
    -- 变更信息
    change_type VARCHAR(50),             -- CREATE, MODIFY, REVIEW, APPROVE
    change_description TEXT,
    
    -- 审核信息
    reviewer_id BIGINT REFERENCES sys_user(id),
    review_status VARCHAR(20),           -- PENDING, APPROVED, REJECTED
    review_comment TEXT,
    reviewed_at TIMESTAMP,
    
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE(document_id, version_number)
);

-- 文档标签表
CREATE TABLE doc_tag (
    id BIGSERIAL PRIMARY KEY,
    tag_name VARCHAR(50) NOT NULL UNIQUE,
    tag_color VARCHAR(20),
    usage_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

-- 文档标签关联表
CREATE TABLE doc_document_tag (
    id BIGSERIAL PRIMARY KEY,
    document_id BIGINT NOT NULL REFERENCES doc_document(id),
    tag_id BIGINT NOT NULL REFERENCES doc_tag(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(document_id, tag_id)

-- 文档访问记录表
CREATE TABLE doc_access_log (
    id BIGSERIAL PRIMARY KEY,
    document_id BIGINT NOT NULL REFERENCES doc_document(id),
    user_id BIGINT NOT NULL REFERENCES sys_user(id),
    action VARCHAR(20) NOT NULL,         -- VIEW, DOWNLOAD, PRINT, SHARE
    ip_address INET,
    user_agent TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
```

## 三、证据管理设计

### 3.1 证据管理概述

证据管理是诉讼案件的核心功能，需要支持：
- 证据收集与登记
- 证据分类与编号
- 证据目录生成
- 质证意见记录
- 证据清单打印

### 3.2 证据分类

```
证据类型（民事诉讼）
├── 书证
│   ├── 合同协议
│   ├── 票据凭证
│   ├── 书信函件
│   └── 其他书面材料
├── 物证
├── 视听资料
│   ├── 录音
│   ├── 录像
│   └── 电子数据
├── 证人证言
├── 当事人陈述
├── 鉴定意见
│   ├── 司法鉴定
│   ├── 审计报告
│   └── 评估报告
└── 勘验笔录

证据类型（刑事诉讼）
├── 物证
├── 书证
├── 证人证言
├── 被害人陈述
├── 犯罪嫌疑人/被告人供述和辩解
├── 鉴定意见
├── 勘验、检查、辨认、侦查实验等笔录
└── 视听资料、电子数据
```

### 3.3 证据管理表结构

```sql
-- 证据表
CREATE TABLE evidence (
    id BIGSERIAL PRIMARY KEY,
    evidence_no VARCHAR(50) NOT NULL,    -- 证据编号（案件内唯一）
    matter_id BIGINT NOT NULL REFERENCES matter(id),
    
    -- 证据基本信息
    evidence_name VARCHAR(200) NOT NULL,
    evidence_type VARCHAR(50) NOT NULL,  -- DOCUMENTARY(书证), PHYSICAL(物证), AUDIO_VIDEO(视听资料), WITNESS(证人证言), EXPERT(鉴定意见), etc.
    evidence_subtype VARCHAR(50),        -- 子类型
    
    -- 证据内容
    description TEXT,                    -- 证据说明
    proof_purpose TEXT NOT NULL,         -- 证明目的
    proof_content TEXT,                  -- 证明内容
    
    -- 证据来源
    source_type VARCHAR(50),             -- ORIGINAL(原件), COPY(复印件), PHOTO(照片), EXTRACT(摘录)
    source_description TEXT,             -- 来源说明
    provider VARCHAR(100),               -- 提供人
    obtain_date DATE,                    -- 取得日期
    obtain_method VARCHAR(100),          -- 取得方式
    
    -- 证据属性
    evidence_direction VARCHAR(20),      -- PLAINTIFF(原告/控方), DEFENDANT(被告/辩方), COURT(法院调取)
    is_original BOOLEAN DEFAULT FALSE,   -- 是否原件
    page_count INT,                      -- 页数
    
    -- 关联文档
    document_id BIGINT REFERENCES doc_document(id),
    
    -- 质证信息
    cross_exam_status VARCHAR(20),       -- PENDING(待质证), COMPLETED(已质证)
    
    -- 法院认定
    court_adoption VARCHAR(20),          -- ADOPTED(采纳), PARTIAL(部分采纳), REJECTED(不采纳), PENDING(待定)
    court_comment TEXT,                  -- 法院意见
    
    -- 排序
    sort_order INT DEFAULT 0,
    group_name VARCHAR(100),             -- 证据组名称（用于分组展示）
    
    -- 状态
    status VARCHAR(20) DEFAULT 'ACTIVE',
    
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_by BIGINT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted BOOLEAN DEFAULT FALSE,
    
    UNIQUE(matter_id, evidence_no)
);

-- 质证意见表
CREATE TABLE evidence_cross_examination (
    id BIGSERIAL PRIMARY KEY,
    evidence_id BIGINT NOT NULL REFERENCES evidence(id),
    matter_id BIGINT NOT NULL REFERENCES matter(id),
    
    -- 质证方
    exam_party VARCHAR(20) NOT NULL,     -- PLAINTIFF, DEFENDANT, THIRD_PARTY
    exam_party_name VARCHAR(100),
    
    -- 质证意见
    authenticity_opinion VARCHAR(20),    -- CONFIRM(认可), DENY(否认), UNKNOWN(不清楚)
    authenticity_reason TEXT,            -- 真实性意见
    legality_opinion VARCHAR(20),
    legality_reason TEXT,                -- 合法性意见
    relevance_opinion VARCHAR(20),
    relevance_reason TEXT,               -- 关联性意见
    
    -- 综合意见
    overall_opinion TEXT,
    
    -- 庭审记录
    court_session_date DATE,             -- 质证日期
    
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 证据清单表（用于生成证据目录）
CREATE TABLE evidence_list (
    id BIGSERIAL PRIMARY KEY,
    list_no VARCHAR(50) NOT NULL UNIQUE,
    matter_id BIGINT NOT NULL REFERENCES matter(id),
    
    -- 清单信息
    list_name VARCHAR(200) NOT NULL,     -- 清单名称
    list_type VARCHAR(50),               -- SUBMIT(提交清单), EXCHANGE(交换清单), COURT(庭审清单)
    submit_date DATE,                    -- 提交日期
    submit_to VARCHAR(200),              -- 提交对象（法院名称）
    
    -- 包含的证据
    evidence_ids JSONB,                  -- 证据ID列表及顺序
    
    -- 生成的文档
    generated_document_id BIGINT REFERENCES doc_document(id),
    
    remarks TEXT,
    
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 四、档案管理设计

### 4.1 档案管理概述

案件结案后需要进行档案整理和归档，档案管理需要支持：
- 档案整理与装订
- 档案入库与定位
- 档案借阅与归还
- 档案保管期限
- 档案销毁流程

### 4.2 档案分类与保管期限

| 档案类别 | 保管期限 | 说明 |
|----------|----------|------|
| 刑事案件 | 永久 | 所有刑事案件档案 |
| 重大民商事案件 | 30年 | 标的额500万以上 |
| 一般民商事案件 | 15年 | 标的额500万以下 |
| 行政案件 | 15年 | |
| 非诉项目 | 10年 | 常年顾问、专项服务等 |
| 咨询类 | 5年 | 简单咨询、法律意见 |

### 4.3 档案管理表结构


### 5.2 模板生成服务

```java
// 模板变量解析服务
@Service
public class TemplateVariableService {
    
    /**
     * 解析模板变量，生成文档
     */
    public byte[] generateDocument(Long templateId, Long matterId, Map<String, Object> extraVariables) {
        // 1. 获取模板
        DocTemplate template = templateRepository.findById(templateId).orElseThrow();
        
        // 2. 收集变量值
        Map<String, Object> variables = collectVariables(matterId);
        variables.putAll(extraVariables);  // 合并额外变量
        
        // 3. 读取模板文件
        byte[] templateBytes = minioService.getFile(template.getFileUrl());
        
        // 4. 替换变量（支持Word和PDF）
        byte[] result;
        if (template.getFileFormat().equals("docx")) {
            result = processWordTemplate(templateBytes, variables);
        } else if (template.getFileFormat().equals("pdf")) {
            result = processPdfTemplate(templateBytes, variables);
        } else {
            throw new BusinessException("不支持的模板格式");
        }
        
        return result;
    }
    
    /**
     * 收集案件相关的所有变量值
     */
    private Map<String, Object> collectVariables(Long matterId) {
        Map<String, Object> variables = new HashMap<>();
        
        // 获取案件信息
        Matter matter = matterRepository.findById(matterId).orElseThrow();
        variables.put("matter.name", matter.getMatterName());
        variables.put("matter.no", matter.getMatterNo());
        
        // 获取客户信息
        Client client = clientRepository.findById(matter.getClientId()).orElseThrow();
        variables.put("client.name", client.getClientName());
        variables.put("client.legalPerson", client.getLegalPerson());
        variables.put("client.address", client.getBusinessAddress());
        
        // 获取律师信息
        User lawyer = userRepository.findById(matter.getMainLawyerId()).orElseThrow();
        variables.put("lawyer.name", lawyer.getRealName());
        variables.put("lawyer.licenseNo", lawyer.getLawyerLicenseNo());
        
        // 获取诉讼扩展信息
        MatterLitigationExt ext = litigationExtRepository.findByMatterId(matterId).orElse(null);
        if (ext != null) {
            variables.put("matter.caseNo", ext.getCaseNo());
            variables.put("matter.caseCause", ext.getCaseCause());
            variables.put("court.name", ext.getCourtName());
        }
        
        // 日期变量
        variables.put("date.today", LocalDate.now().format(DateTimeFormatter.ofPattern("yyyy年MM月dd日")));
        variables.put("date.year", String.valueOf(LocalDate.now().getYear()));
        
        // 律所信息（从系统配置获取）
        variables.put("firm.name", configService.get("firm.name"));
        variables.put("firm.address", configService.get("firm.address"));
        
        return variables;
    }
}
```

## 九、总结

本补充方案针对原方案中文档、证据、档案管理的不足，进行了以下增强：

1. **文档管理**：增加了专业的文档分类体系、文档标签、访问日志
2. **证据管理**：新增完整的证据管理模块，支持证据登记、质证记录、清单生成

4. **模板系统**：增强了模板变量系统，支持自动填充案件信息

这些功能对于诉讼业务为主的律所尤为重要，建议在核心功能上线后优先实施。
