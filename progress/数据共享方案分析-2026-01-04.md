# 律师业务、财务、行政数据共享方案

> 日期：2026-01-04

---

## 核心原则：一切围绕项目

```
┌─────────────────────────────────────────────────────────────────┐
│                      业务数据流转链路                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   客户 ──────> 合同 ──────> 项目 ──────> 财务/行政操作           │
│   (律师创建)   (需审批)    (核心)     (基于项目)                │
│                                                                 │
│   没有项目，财务和行政就没有操作基础                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 一、数据依赖关系

### 1.1 项目是中心

| 前置条件 | 说明 |
|----------|------|
| 客户 | 必须先有客户，才能签合同 |
| 合同 | 必须有合同且审批通过，才能立项目 |
| 项目 | 有了项目，财务和行政才能基于项目操作 |

### 1.2 各模块基于项目的操作

| 模块 | 基于项目的操作 | 说明 |
|------|----------------|------|
| **财务** | 收款登记 | 选择项目，登记该项目的收款 |
| **财务** | 费用报销 | 选择项目，归集该项目的成本 |
| **财务** | 提成计算 | 基于项目团队分配提成 |
| **财务** | 开票 | 基于项目/合同开票 |
| **行政** | 介绍信 | 选择项目，为该项目律师出具介绍信 |
| **行政** | 公函 | 选择项目/客户，出具公函 |

### 1.3 主任/合伙人的视角

| 权限 | 说明 |
|------|------|
| 查看所有项目 | 了解全所业务情况 |
| 查看所有财务数据 | 了解收入、成本、利润 |
| 审批合同/项目 | 控制业务准入 |
| 审批费用报销 | 控制成本支出 |

## 三、数据权限矩阵

### 3.1 按角色划分

| 数据 | 律师 | 财务 | 行政 | 主任/合伙人 |
|------|------|------|------|-------------|
| 客户 | 创建/编辑 | 只读 | 只读 | 全部 |
| 合同 | 创建/编辑 | 只读 | - | 全部+审批 |
| 项目 | 创建/编辑(自己的) | 只读 | 只读 | 全部+审批 |
| 收款 | 只读(自己项目) | 创建/编辑 | - | 全部 |
| 费用 | 申请 | 审核/支付 | - | 全部+审批 |
| 提成 | 只读(自己的) | 计算/发放 | - | 全部 |
| 介绍信 | 申请 | - | 出具 | 全部 |

### 3.2 数据范围

| 角色 | 数据范围 |
|------|----------|
| 普通律师 | 只能看自己参与的项目及相关数据 |
| 财务人员 | 可以看所有项目的财务数据 |
| 行政人员 | 可以看所有项目的基本信息 |
| 主任/合伙人 | 可以看所有数据 |
