#### 问题1: 更新证据接口参数格式不匹配

**前端实现** (`frontend/apps/web-antd/src/api/evidence/index.ts:92-94`):
```typescript
export function updateEvidence(id: number, data: UpdateEvidenceCommand) {
  return requestClient.put<EvidenceDTO>(`/evidence/${id}`, data);
}
```
- 使用 PUT 请求体传递 `UpdateEvidenceCommand` 对象

**后端实现** (`backend/src/main/java/com/lawfirm/interfaces/rest/evidence/EvidenceController.java:60-77`):
```java
@PutMapping("/{id}")
public Result<EvidenceDTO> update(@PathVariable Long id,
                                  @RequestParam(required = false) String name,
                                  @RequestParam(required = false) String evidenceType,
                                  // ... 多个 @RequestParam 参数
                                  ) {
    // ...
}
```
- 使用多个 `@RequestParam` 参数，期望从 URL 查询参数获取

**影响**: ❌ 前端无法成功调用后端更新接口

**解决方案**: 
1. **推荐方案**: 修改后端，使用 `@RequestBody UpdateEvidenceCommand` 接收参数
2. 备选方案: 修改前端，将数据转换为 URL 查询参数（不推荐，参数太多）

### 2. 系统管理模块 (System)

#### 问题1: 字典管理接口路径不一致 ⚠️

**前端实现** (`frontend/apps/web-antd/src/api/system/index.ts`):
```typescript
// 获取字典类型列表
export function getDictTypeList() {
  return requestClient.get<DictTypeDTO[]>('/system/dict/type/list', { params });
}

// 获取字典数据
export function getDictDataByCode(dictCode: string) {
  return requestClient.get<DictDataDTO[]>(`/system/dict/data/${dictCode}`);
}
```

**后端实现** (`backend/src/main/java/com/lawfirm/interfaces/rest/system/DictController.java`):
```java
// 获取所有字典类型
@GetMapping("/types")
public Result<List<DictTypeDTO>> listDictTypes() {
    // 路径: /system/dict/types
}

// 根据编码获取字典项
@GetMapping("/items/code/{code}")
public Result<List<DictItemDTO>> getDictItemsByCode(@PathVariable String code) {
    // 路径: /system/dict/items/code/{code}
}
```

**问题**:
- 前端: `/system/dict/type/list` → 后端: `/system/dict/types` ❌
- 前端: `/system/dict/data/{dictCode}` → 后端: `/system/dict/items/code/{code}` ❌

**解决方案**: 需要统一路径，建议修改前端API路径以匹配后端。
