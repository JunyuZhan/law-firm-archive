# 🏢 智慧律所一体化管理平台 - 完整本地部署方案

## 📋 **系统全景概览**

### **核心设计理念**
1. **本地化全栈部署**：所有服务部署在律所内部服务器，数据不出内网
2. **模块化微服务架构**：松耦合，便于维护和扩展
3. **全业务覆盖**：从客户咨询到案件归档，全流程电子化管理
4. **智能化辅助**：集成AI能力提升工作效率
5. **安全合规**：符合律师行业数据安全要求

    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./init-scripts:/docker-entrypoint-initdb.d
    ports:
      - "5432:5432"
    networks:
      - law_network
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U law_admin"]
      interval: 10s
      timeout: 5s
      retries: 5

  postgres-slave:
    image: postgres:15-alpine
    container_name: postgres-slave
    restart: always
    environment:
      POSTGRES_DB: law_firm
      POSTGRES_USER: law_admin
      POSTGRES_PASSWORD: ${DB_PASSWORD}

      - "9000:9000"   # API端口

      - law_network

      test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
      interval: 30s
      timeout: 20s
      retries: 3

    image: minio/mc:latest

    restart: on-failure

      mc mb law-minio/documents &&

### **2.3 Nginx配置文件**

```nginx
# nginx/nginx.conf
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;
    multi_accept on;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;
    client_max_body_size 100M;

    # 开启gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript 
               application/javascript application/xml+rss 
               application/json;

    # SSL配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # 上游服务配置
    upstream backend_servers {
        least_conn;
        server gateway-service:8080 max_fails=3 fail_timeout=30s;
        keepalive 32;
    }

    upstream onlyoffice_server {
        server onlyoffice:80;
    }

    upstream paddle_ocr_server {
        server paddle-ocr:8000;
    }

    # 主服务器配置
    server {
        listen 80;
        server_name lawfirm.local;
        
        # HTTP重定向到HTTPS
        location / {
            return 301 https://$server_name$request_uri;
        }
    }

    server {
        listen 443 ssl http2;
        server_name lawfirm.local;

        # SSL证书配置（使用自签名证书，生产环境替换为正式证书）
        ssl_certificate /etc/nginx/ssl/lawfirm.crt;
        ssl_certificate_key /etc/nginx/ssl/lawfirm.key;

        # 安全头
        add_header X-Frame-Options SAMEORIGIN;
        add_header X-Content-Type-Options nosniff;
        add_header X-XSS-Protection "1; mode=block";
        add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

        # 前端静态资源
        location / {
            root /usr/share/nginx/html;
            index index.html index.htm;
            try_files $uri $uri/ /index.html;
            
            # 缓存策略
            location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
                expires 1y;
                add_header Cache-Control "public, immutable";
            }
            
            location ~* \.(html)$ {
                expires 1h;
                add_header Cache-Control "public, must-revalidate";
            }
        }

        # API代理
        location /api/ {
            proxy_pass http://backend_servers;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # 超时设置
            proxy_connect_timeout 300s;
            proxy_send_timeout 300s;
            proxy_read_timeout 300s;
            
            # 缓冲区设置
            proxy_buffering on;
            proxy_buffer_size 4k;
            proxy_buffers 8 4k;
            proxy_busy_buffers_size 8k;
        }

        # WebSocket支持（用于实时通知）
        location /ws/ {
            proxy_pass http://backend_servers;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "Upgrade";
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            
            # WebSocket超时设置
            proxy_read_timeout 86400s;
            proxy_send_timeout 86400s;
        }

        # OnlyOffice文档服务代理
        location /office/ {
            proxy_pass http://onlyoffice_server/;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Forwarded-Host $host;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            
            # 文件上传大小限制

        }

            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

            proxy_set_header X-Real-IP $remote_addr;

            proxy_set_header X-Forwarded-Proto $scheme;
            
            # 文件下载缓存
            proxy_cache my_cache;

            proxy_cache_valid 200 302 10m;

            client_max_body_size 500M;
        }

        # 管理界面代理（开发环境使用）
        location /nacos/ {

        }

        location /rabbitmq/ {

            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            auth_basic "Restricted Access";
            auth_basic_user_file /etc/nginx/.htpasswd;
        }

        # 健康检查端点
        location /health {
            access_log off;
            return 200 "healthy\n";
            add_header Content-Type text/plain;
        }
    }

    # 缓存配置
    proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m inactive=60m use_temp_path=off;

## 📝 **七、方案完善意见**

### **7.1 架构简化建议**

#### 问题：微服务架构过度设计

20-50人规模的律所，微服务带来的复杂度远超收益：
- 10+ 服务的部署、监控、排错成本高
- 中间件（Nacos、RabbitMQ、Gateway）本身占用 10-20G 内存
- 分布式事务处理复杂
- 运维团队配置要求高

#### 建议：单体 + 模块化架构

```
law-firm-system/
├── src/main/java/com/lawfirm/
│   ├── modules/
│   │   ├── auth/           # 认证授权模块
│   │   ├── client/         # 客户管理模块
│   │   ├── project/        # 项目管理模块
│   │   ├── finance/        # 财务管理模块
│   │   ├── document/       # 文书管理模块
│   │   ├── workflow/       # 工作流模块
│   │   ├── knowledge/      # 知识库模块
│   │   └── admin/          # 行政办公模块
│   ├── common/             # 公共组件
│   │   ├── config/         # 配置类
│   │   ├── security/       # 安全组件
│   │   ├── exception/      # 异常处理
│   │   └── util/           # 工具类
│   └── LawFirmApplication.java
├── src/main/resources/
│   ├── application.yml
│   └── mapper/             # MyBatis映射文件
└── pom.xml
```

#### 简化后的技术栈

| 组件 | 原方案 | 简化方案 | 节省资源 |
|------|--------|----------|----------|
| 应用框架 | Spring Cloud 微服务 | Spring Boot 单体 | 内存 -20G |
| 服务注册 | Nacos | 不需要 | 内存 -2G |
| 配置中心 | Nacos | application.yml + 数据库配置表 | 简化运维 |
| API网关 | Spring Cloud Gateway | Nginx 反向代理 | 内存 -1G |
| 消息队列 | RabbitMQ | 数据库队列表 / Redis Stream | 内存 -1G |
| 任务调度 | XXL-Job | Spring @Scheduled + ShedLock | 简化部署 |

#### 问题

**方案一：预览为主 + 下载编辑（推荐）**

- 符合律师实际工作习惯（本地编辑更流畅）
- 版本管理更清晰

### **7.8 总结**

本方案的核心设计思路是正确的，业务模块划分合理，数据库设计完整。主要问题在于**技术架构过度设计**，不适合 20-50 人规模的律所。

建议采用**单体 + 模块化**架构，保持代码结构清晰的同时大幅降低开发和运维成本。等业务规模真正增长到需要微服务时，再进行拆分，这时候模块化的代码结构会让拆分变得相对容易。

**核心原则：够用就好，预留扩展能力。**
