# 文档1：项目启动与环境配置指南

## 概述

本文档旨在指导开发团队完成智慧律所管理系统的开发环境搭建、项目初始化和基础框架配置。预计完成时间：5-7个工作日。

## 一、环境准备清单

### 1.1 硬件要求

| 设备类型 | 最低配置 | 推荐配置 | 备注 |
|----------|----------|----------|------|
| 开发机 | 8GB内存，256GB SSD | 16GB内存，512GB SSD | 用于日常开发 |
| 测试服务器 | 16GB内存，500GB SSD | 32GB内存，1TB SSD | Docker运行环境 |
| 网络 | 稳定互联网连接 | 100Mbps以上带宽 | 依赖包下载 |

## 三、Docker环境配置

### 3.1 开发环境docker-compose.yml

```yaml
# docker-compose.yml
version: '3.8'

services:
  # PostgreSQL 数据库
  postgres:
    image: postgres:15-alpine
    container_name: law-postgres-dev
    restart: unless-stopped
    environment:
      POSTGRES_DB: law_firm_dev
      POSTGRES_USER: law_admin
      POSTGRES_PASSWORD: dev_password_123
      PGDATA: /var/lib/postgresql/data/pgdata
      TZ: Asia/Shanghai
    volumes:
      - postgres_data_dev:/var/lib/postgresql/data
      - ./init-db:/docker-entrypoint-initdb.d
    ports:
      - "5432:5432"
    networks:
      - law-network-dev
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U law_admin"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: law-redis-dev
    restart: unless-stopped
    command: >
      redis-server
      --requirepass dev_redis_123
      --appendonly yes
      --maxmemory 256mb
      --maxmemory-policy allkeys-lru
    volumes:
      - redis_data_dev:/data
    ports:
      - "6379:6379"
    networks:
      - law-network-dev
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "dev_redis_123", "ping"]
      interval: 10s
      timeout: 5s
      retries: 3

  # MinIO 对象存储
  minio:
    image: minio/minio:RELEASE.2024-01-18T22-51-09Z
    container_name: law-minio-dev
    restart: unless-stopped
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: admin
      MINIO_ROOT_PASSWORD: dev_minio_123
    volumes:
      - minio_data_dev:/data
    ports:
      - "9000:9000"
      - "9001:9001"
    networks:
      - law-network-dev
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
      interval: 30s
      timeout: 10s
      retries: 3

  # Elasticsearch 全文检索
  elasticsearch:
    image: elasticsearch:8.11.0
    container_name: law-elasticsearch-dev
    restart: unless-stopped
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
      - cluster.name=law-firm-dev
    volumes:
      - elasticsearch_data_dev:/usr/share/elasticsearch/data
    ports:
      - "9200:9200"
    networks:
      - law-network-dev
    ulimits:
      memlock:
        soft: -1
        hard: -1
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9200/_cluster/health"]
      interval: 30s
      timeout: 10s
      retries: 5

volumes:
  postgres_data_dev:
  redis_data_dev:
  minio_data_dev:
  elasticsearch_data_dev:

networks:
  law-network-dev:
    driver: bridge
```

## 八、常见问题解决

### 8.1 Docker启动问题

**问题1：端口被占用**
```bash
# 查看端口占用
netstat -tlnp | grep :5432

# 停止占用进程或修改docker-compose端口映射
# 修改为：
ports:
  - "5433:5432"  # 外部端口:内部端口
```

**问题2：权限不足**
```bash
# 使用sudo运行
sudo docker-compose up -d

# 或将用户加入docker组
sudo usermod -aG docker $USER
```

### 8.2 数据库连接问题

**问题：连接被拒绝**
```bash
# 检查PostgreSQL是否允许远程连接
# 修改pg_hba.conf，添加：
host    all             all             0.0.0.0/0            md5

# 重启PostgreSQL
docker restart law-postgres-dev
```

### 8.3 前端依赖安装问题

**问题：pnpm安装慢或失败**
```bash
# 清除缓存并重试
pnpm store prune
pnpm install --force

# 或使用npm
npm config set registry https://registry.npmmirror.com
npm install
```

### A. 常用命令速查

```bash
# Docker相关
docker-compose up -d              # 启动所有服务
docker-compose down              # 停止所有服务
docker-compose logs -f service   # 查看服务日志
docker-compose ps                # 查看服务状态

# 后端相关
mvn clean compile               # 清理并编译
mvn spring-boot:run             # 运行Spring Boot应用
mvn test                        # 运行测试
mvn package                     # 打包应用

# 前端相关
pnpm dev                        # 启动开发服务器
pnpm build                      # 构建生产版本
pnpm lint                       # 代码检查
pnpm type-check                 # TypeScript类型检查
```
