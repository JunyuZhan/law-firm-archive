# 前端后端对接指南

## 一、环境配置

### 1.1 后端服务
- **地址**: `http://localhost:8080`
- **API路径**: `/api`
- **完整地址**: `http://localhost:8080/api`
- **状态**: ✅ 已启动

### 1.2 前端服务
- **开发端口**: `5555` (默认)
- **API代理**: `/api` → `http://localhost:8080`
- **配置位置**: `frontend/apps/web-antd/vite.config.mts`

### 1.3 环境变量配置

在 `frontend/apps/web-antd/` 目录下创建或检查 `.env.development` 文件：

```bash
# 端口号
VITE_PORT=5555

# 资源公共路径
VITE_BASE=/

# 接口地址（开发环境使用代理）
VITE_GLOB_API_URL=/api

# 是否开启 Nitro Mock服务（对接后端时设为false）
VITE_NITRO_MOCK=false

# 是否打开 devtools
VITE_DEVTOOLS=true

# 是否注入全局loading
VITE_INJECT_APP_LOADING=true
```

## 二、API响应格式对接

### 2.1 后端响应格式

后端统一使用 `Result<T>` 格式：

```json
{
  "success": true,
  "code": "200",
  "message": "操作成功",
  "data": { ... },
  "timestamp": 1767446015461
}
```

### 2.2 前端响应拦截器配置

已配置在 `frontend/apps/web-antd/src/api/request.ts`：

```typescript
client.addResponseInterceptor(
  defaultResponseInterceptor({
    codeField: 'success',        // 成功标识字段
    dataField: 'data',            // 数据字段
    successCode: (code: any) => code === true,  // 成功判断逻辑
  }),
);
```

### 2.3 错误处理

后端错误响应格式：
```json
{
  "success": false,
  "code": "500",
  "message": "错误信息",
  "data": null,
  "timestamp": 1767446015461
}
```

前端错误拦截器会自动提取 `message` 字段显示错误信息。

## 三、认证接口对接

### 3.1 登录接口

**后端接口**: `POST /api/auth/login`

**请求参数**:
```typescript
{
  username: string;
  password: string;
  captchaId?: string;    // 验证码ID（可选）
  captchaCode?: string;   // 验证码（可选）
}
```

**响应数据**:
```typescript
{
  accessToken: string;
  refreshToken: string;
  expiresIn: number;     // 过期时间（秒）
  userId: number;
  username: string;
  realName: string;
  roles: string[];
  permissions: string[];
}
```

**前端调用**:
```typescript
import { loginApi } from '#/api/core/auth';

const result = await loginApi({
  username: 'admin',
  password: 'admin123',
  captchaId: 'xxx',
  captchaCode: '1234'
});
```

### 3.2 获取验证码

**后端接口**: `GET /api/auth/captcha`

**响应数据**:
```typescript
{
  captchaId: string;
  captchaImage: string;  // Base64图片
}
```

**前端调用**:
```typescript
import { getCaptchaApi } from '#/api/core/auth';

const result = await getCaptchaApi();
```

### 3.3 刷新Token

**后端接口**: `POST /api/auth/refresh`

**请求参数**:
```typescript
{
  refreshToken: string;
}
```

**响应数据**: 同登录接口

**前端调用**:
```typescript
import { refreshTokenApi } from '#/api/core/auth';

const result = await refreshTokenApi();
```

### 3.4 获取用户信息

**后端接口**: `GET /api/auth/info`

**响应数据**:
```typescript
{
  userId: number;
  username: string;
  realName: string;
  departmentId?: number;
  compensationType?: string;
  roles: string[];
  permissions: string[];
}
```

**前端调用**:
```typescript
import { getUserInfoApi } from '#/api/core/auth';

const userInfo = await getUserInfoApi();
```

### 3.5 登出接口

**后端接口**: `POST /api/auth/logout`

**前端调用**:
```typescript
import { logoutApi } from '#/api/core/auth';

await logoutApi();
```

## 六、常见问题

### 6.1 跨域问题

**问题**: 前端请求出现 CORS 错误

**解决**: 
- 检查后端 `CorsConfig` 配置
- 确认前端代理配置正确（`vite.config.mts`）

### 6.2 Token过期

**问题**: 401 Unauthorized

**解决**: 
- 检查前端是否自动刷新Token
- 检查 `refreshTokenApi` 实现
- 检查后端Token验证逻辑

### 6.3 响应格式不匹配

**问题**: 前端无法解析响应数据

**解决**: 
- 检查 `defaultResponseInterceptor` 配置
- 确认 `codeField` 和 `dataField` 与后端一致
- 检查 `successCode` 判断逻辑

### 6.4 代理不生效

**问题**: 前端请求404或无法连接

**解决**: 
- 检查 `vite.config.mts` 中的 `proxy` 配置
- 确认后端服务已启动
- 检查端口是否被占用

## 九、开发建议

### 9.1 接口调用规范
1. 使用 `requestClient` 进行所有API调用
2. 统一使用 TypeScript 类型定义
3. 错误处理统一在拦截器中处理
4. 业务逻辑错误通过 `Result.message` 传递

### 9.2 类型定义
- 所有接口参数和返回值都应定义TypeScript类型
- 类型定义放在对应的API文件中
- 使用命名空间组织相关类型

### 9.3 错误处理
- 网络错误：自动显示错误提示
- 业务错误：显示后端返回的 `message`
- Token过期：自动刷新或跳转登录
