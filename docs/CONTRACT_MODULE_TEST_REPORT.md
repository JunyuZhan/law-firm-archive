## 已验证的业务规则

### 合同状态流转

```
DRAFT (草稿)
  ├── 可以编辑
  ├── 可以删除
  └── 可以提交审批 → PENDING

PENDING (待审批)
  ├── 不能编辑
  ├── 不能删除
  ├── 不能重复提交
  └── 等待审批处理

ACTIVE (生效中)
  ├── 不能直接编辑（需要变更审批）
  ├── 不能删除
  └── 可以申请变更

REJECTED (已拒绝)
  ├── 可以编辑
  └── 可以重新提交审批
```

### 数据完整性约束

1. **合同必须关联客户** - 创建合同时clientId为必填项
2. **自动生成合同编号** - 系统根据案件类型和收费类型自动生成规范编号
3. **风险比例范围限制** - 必须在0-100之间
4. **提成比例限制** - 所有参与人的提成比例总和不能超过100%
5. **主办律师自动添加** - 创建合同时自动将签约人添加为主办律师

### 权限控制

1. **数据范围控制** - 用户只能看到自己有权限的合同
2. **编辑权限** - 只有合同创建者、签约人或参与人可以编辑
3. **删除权限** - 只有草稿状态的合同可以删除
