Prometheus 中文使用指南


概述

Prometheus 界面是英文的，但功能很强大。本指南帮你快速上手使用。

推荐：如果觉得 Prometheus 英文界面不方便，建议使用 Grafana（支持中文），它可以从 Prometheus 读取数据并以更友好的方式展示。


方法一：使用浏览器翻译（最简单）

Chrome/Edge 浏览器

1. 打开 Prometheus 页面
2. 右键点击页面任意位置
3. 选择"翻译为中文" 或 "Translate to Chinese"
4. 界面会自动翻译为中文

操作步骤

访问 Prometheus → 右键 → 翻译为中文 → 完成 ✅


方法二：使用 Grafana（推荐，支持中文）

Grafana 支持中文界面，并且可以显示 Prometheus 的所有数据。

快速步骤

1. 访问 Grafana
   http://your-server-ip:3000

2. 登录
   - 用户名：admin
   - 密码：查看 .env 文件中的 GRAFANA_PASSWORD

3. 切换到中文
   - 点击左下角用户图标
   - 选择 "Preferences"（偏好设置）
   - 在 "Language" 中选择 "中文（简体）"
   - 点击 "Save"（保存）

4. 查看监控数据
   - 左侧菜单 → "仪表板" → "浏览"
   - 点击 "Spring Boot Overview" 查看后端监控

常用查询示例

1. 检查服务是否正常运行

查询语句：
up{job="law-firm-backend"}

操作步骤：
1. 在 "Enter expression" 输入框输入上面的查询语句
2. 点击 "Execute"（执行）按钮
3. 查看结果：
   - 值为 1 = 服务正常 ✅
   - 值为 0 = 服务异常 ❌

查看图表：
- 点击 "Graph" 标签，可以看到历史趋势


2. 查看所有服务状态

查询语句：
up

说明：显示所有被监控服务的状态


3. 查看内存使用情况

查询语句：
jvm_memory_used_bytes{application="law-firm-management"} / 1024 / 1024

说明：
- 显示已使用内存（单位：MB）
- application="law-firm-management" 是后端服务的标识

查看图表：
- 点击 "Graph" 标签，可以看到内存使用趋势


4. 查看 HTTP 请求总数

查询语句：
http_server_requests_seconds_count

说明：显示所有 HTTP 请求的总数


5. 查看请求速率（每秒请求数）

查询语句：
rate(http_server_requests_seconds_count[5m])

说明：
- 显示最近 5 分钟的平均请求速率
- 单位：请求/秒


6. 查看平均响应时间

查询语句：
http_server_requests_seconds_sum / http_server_requests_seconds_count

说明：显示平均响应时间（单位：秒）


7. 查看数据库连接池

查询语句：
hikari_connections_active

说明：显示数据库活跃连接数


8. 查看 CPU 使用率

查询语句：
system_cpu_usage

说明：显示系统 CPU 使用率（0-1 之间，1 = 100%）


操作步骤详解

步骤 1：打开查询页面

1. 访问 Prometheus：http://your-server-ip:9090
2. 顶部导航栏默认就是 "Query"（查询）页面

步骤 2：输入查询语句

1. 在 "Enter expression" 输入框中输入查询语句
2. 例如：up{job="law-firm-backend"}

步骤 3：执行查询

1. 点击 "Execute"（执行）按钮
2. 或者按 Enter 键

步骤 4：查看结果

表格视图（Table）：
- 默认显示为表格
- 显示当前值和时间戳

图表视图（Graph）：
- 点击 "Graph" 标签
- 显示历史趋势图表
- 可以调整时间范围

解释视图（Explain）：
- 点击 "Explain" 标签
- 显示查询语句的解析过程


查看监控目标（Targets）

操作步骤

1. 点击顶部导航栏的 "Status"
2. 在下拉菜单中选择 "Targets"
3. 查看所有被监控的服务

状态说明

- 绿色（UP） = 服务正常 ✅
- 红色（DOWN） = 服务异常 ❌

应该看到的目标

- law-firm-backend - 后端服务监控


常用监控场景

场景 1：检查服务是否正常

查询：
up{job="law-firm-backend"}

结果判断：
- 1 = 正常
- 0 = 异常


场景 2：检查内存是否充足

查询：
jvm_memory_used_bytes{application="law-firm-management"} / 1024 / 1024

查看图表：
- 点击 "Graph" 查看趋势
- 如果持续上升，可能需要关注


场景 3：检查 API 性能

查询请求速率：
rate(http_server_requests_seconds_count[5m])

查询响应时间：
http_server_requests_seconds_sum / http_server_requests_seconds_count

查看图表：
- 点击 "Graph" 查看历史趋势
- 如果响应时间突然增加，可能有性能问题


场景 4：检查数据库连接

查询：
hikari_connections_active

说明：
- 显示当前活跃的数据库连接数
- 如果接近最大值，可能需要优化
