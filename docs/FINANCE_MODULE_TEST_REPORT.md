## 5. 问题修复记录

### 5.1 金额校验缺失问题

**问题描述**: 创建收款记录时，系统未校验金额是否为正数，允许负数和零金额

**影响范围**: 
- `CreatePaymentCommand` - 收款命令
- `CreateFeeCommand` - 收费命令
- `CreateInvoiceCommand` - 发票命令

**修复方案**: 为相关 Command 类的 `amount` 字段添加 `@Positive` 注解

**修复代码**:
```java
@NotNull(message = "收款金额不能为空")
@Positive(message = "收款金额必须大于0")
private BigDecimal amount;
```

**修复状态**: ✅ 已修复并验证通过
