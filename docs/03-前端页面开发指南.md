# 前端二次开发指南 (Vben Admin 5.x)

## 概述

本项目基于 [Vben Admin 5.x](https://doc.vben.pro/) 构建，前端开发主要工作为**对接后端接口**与**开发业务页面**。请严格遵循 Vben Admin 5.x 的开发规范与本项目约定。

前端主应用位置：`frontend/apps/web-antd`

## 一、开发流程

完成一个业务模块（如"客户管理"）通常按照以下顺序进行：

1.  **定义 API**: 在 `src/api` 中创建对应模块的接口定义。
2.  **创建页面**: 在 `src/views` 中创建业务页面组件。
3.  **配置路由**: 本项目生产环境使用**后端动态路由**，开发阶段可配置静态路由调试，最终需同步到后端数据库。
4.  **国际化**: 在 `src/locales` 中补充多语言配置。

## 二、目录结构约定

### 2.1 API 目录
所有接口定义位于 `src/api` 目录下，按**业务模块**平铺：

```
src/api/
├── crm/                # 客户/案源相关
├── matter/             # 项目/案件相关
├── finance/            # 财务相关
├── system/             # 系统管理
└── request.ts          # 统一请求封装
```

### 2.2 页面目录
所有页面组件位于 `src/views` 目录下，按**业务领域**分组（与菜单结构一致）：

```
src/views/
├── crm/
│   ├── client/         # 客户管理
│   │   └── index.vue
│   └── lead/           # 案源管理
├── matter/             # 项目管理
└── system/             # 系统管理
```

> **注意**：API 目录倾向于扁平化或按后端服务划分，View 目录倾向于按菜单层级划分。

## 三、API 定义规范

请参照 `src/api/client/index.ts` 的写法。

1.  **路径**: 新建 `src/api/{模块}/index.ts`。
2.  **类型定义**: 在 `types.ts` 中定义 DTO。
3.  **引用**: 使用 `#/api/request` 引入封装好的 `requestClient`。

示例 (`src/api/demo/index.ts`):
```typescript
import { requestClient } from '#/api/request';

export function getDemoList(params: any) {
  return requestClient.get('/demo/list', { params });
}

export function createDemo(data: any) {
  return requestClient.post('/demo', data);
}
```

> **API 前缀说明**: 前端请求不需要加 `/api` 前缀，`requestClient` 内部已配置 `baseURL`，且开发代理 (`vite.config.ts`) 会自动转发。

## 四、页面开发规范

使用 Vue 3 `script setup` 语法与 Vben 5.x 组件库。

### 4.1 常用组件

*   **Page**: 页面容器，提供标题、面包屑等。
    ```vue
    import { Page } from '@vben/common-ui';
    ```
*   **Ant Design Vue**: 基础 UI 组件。
    ```vue
    import { Card, Table, Button, Form, Modal } from 'ant-design-vue';
    ```

### 4.2 典型列表页模板

参考 `src/views/crm/client/index.vue`。

```vue
<script setup lang="ts">
import { ref, reactive, onMounted } from 'vue';
import { Page } from '@vben/common-ui';
import { Card, Table, Button, message } from 'ant-design-vue';
import { getDemoList } from '#/api/demo';

// 定义表格列
const columns = [
  { title: '名称', dataIndex: 'name' },
  { title: '操作', key: 'action' },
];

// 状态管理
const loading = ref(false);
const dataSource = ref([]);
const pagination = reactive({
  current: 1,
  pageSize: 10,
  total: 0,
});

// 加载数据
async function fetchData() {
  loading.value = true;
  try {
    const res = await getDemoList({ 
      pageNum: pagination.current, 
      pageSize: pagination.pageSize 
    });
    dataSource.value = res.list;
    pagination.total = res.total;
  } finally {
    loading.value = false;
  }
}

onMounted(fetchData);
</script>

<template>
  <Page title="示例页面">
    <Card>
      <div class="mb-4">
        <Button type="primary">新增</Button>
      </div>
      <Table 
        :columns="columns" 
        :data-source="dataSource" 
        :loading="loading"
        :pagination="pagination"
      />
    </Card>
  </Page>
</template>
```

## 五、权限与路由

### 5.1 权限控制
后端返回权限码列表 (Access Codes)。

*   **按钮级别**: 使用 `v-access` 指令。
    ```vue
    <Button v-access:code="'client:create'">新增</Button>
    ```

### 5.2 菜单配置
目前采用 **Backend 模式**，菜单由接口 `/system/menu/user` 返回。
若需开发新页面，请联系后端添加菜单数据，或在本地 `mock` 环境模拟。

*   **`$t`**: 国际化函数 `import { $t } from '#/locales'`.
