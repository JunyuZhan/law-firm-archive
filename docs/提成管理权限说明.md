# 提成管理权限说明

## 概述

提成管理是律所的核心财务制度，涉及律师切身利益，必须严格按照权限分级进行管理。系统将提成管理分为两个层面：

1. **提成规则管理**：只有系统管理员和主任可以设定和修改
2. **提成查看和操作**：财务相关角色可以查看和处理日常提成业务

## 权限分级

### 第一级：提成规则管理（仅 ADMIN / DIRECTOR）

这是律所的核心财务制度，只有最高权限角色可以设定：

| 权限 | 功能 | 说明 |
|------|------|------|
| `finance:commission:rule:create` | 创建提成规则 | 新建提成计算规则 |
| `finance:commission:rule:update` | 修改提成规则 | 修改律所留存率、案源提成率等核心参数 |
| `finance:commission:rule:delete` | 删除提成规则 | 删除不再使用的提成规则 |
| `finance:commission:rule:view` | 查看提成规则 | 查看所有提成规则配置 |

**可设置的核心参数：**
- 律所留存比例（这是律所的收入，必须严格管控）
- 案源提成比例（律所统一规定的案源人提成标准）
- 办案提成总比例（律所统一规定的办案人员提成标准）
- 税率设置
- 管理费率设置
- 阶梯费率设置

### 第二级：提成查看和操作（财务角色）

日常提成业务的查看和处理：

| 角色 | 权限 | 说明 |
|------|------|------|
| PARTNER | `finance:commission:view` | 查看所有提成明细和统计 |
| FINANCE_MANAGER | `finance:commission:view`<br>`finance:commission:issue` | 查看提成 + 提成发放 |
| FINANCE | `finance:commission:view`<br>`finance:commission:issue` | 查看提成 + 提成发放 |
| DIRECTOR | 所有权限 | 包括规则管理和日常操作 |
| ADMIN | 所有权限 | 包括规则管理和日常操作 |

### 第三级：个人提成查看（所有律师）

每个律师只能查看自己的提成：

| 权限 | 功能 | 适用角色 |
|------|------|----------|
| `finance:my:commission` | 我的提成 | 所有律师角色 |

## 业务流程控制

### 1. 提成规则设定流程

```
系统管理员/主任 → 提成规则管理 → 设定核心参数 → 保存生效
```

- 只有 ADMIN 和 DIRECTOR 可以访问提成规则管理页面
- 修改后的规则对新产生的收款自动生效
- 历史提成不受影响

### 2. 日常提成处理流程

```
收款到账 → 系统自动计算提成 → 财务查看确认 → 主任审批 → 财务发放
```

- 系统根据设定的规则自动计算提成
- 财务角色可以查看和处理提成发放
- 主任负责提成审批
- 财务确认提成已发放

### 3. 项目级别的分配调整

```
主办律师 → 项目团队管理 → 调整协办律师分配比例 → 保存
```

- 主办律师可以调整办案人员内部的分配比例
- 但不能修改律所留存率、案源提成率等核心参数
- 分配比例总和必须等于100%

## 安全控制

1. **数据库级别**：通过角色权限表严格控制菜单和按钮访问
2. **接口级别**：后端API根据用户角色验证操作权限
3. **前端级别**：根据权限动态显示/隐藏功能按钮
4. **审计日志**：记录所有提成规则的修改操作

## 注意事项

1. **提成规则是律所核心制度**，修改权限必须严格控制
2. **律所留存比例直接影响律所收入**，只能由最高权限角色设定
3. **案源提成比例影响业务拓展激励**，需要统一标准
4. **办案提成比例影响律师收入分配**，需要公平合理
5. **所有规则修改都有审计日志**，确保可追溯
