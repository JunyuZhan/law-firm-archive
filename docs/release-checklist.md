# 发布前验收清单

本文档用于每次版本发布、客户首次部署或升级前的统一核查，避免出现“镜像打好了，但版本、脚本、文档、测试、回滚准备不一致”的情况。

## 1. 版本与源码

- [ ] 已确认本次发布版本号，例如 `v0.1.1`
- [ ] 已更新仓库根目录 **`VERSION`**，并已执行 **`scripts/sync-version.sh`**（同步 `backend/pom.xml` 与 `frontend/package.json`）
- [ ] 已确认 Git 提交号
- [ ] 已完成本次变更说明整理
- [ ] 已确认未误带调试代码、临时开关、测试账号说明

## 2. 镜像与构建

- [ ] 后端镜像已按版本号打标签
- [ ] 前端镜像已按版本号打标签
- [ ] 如有 ES 定制镜像，已按版本号打标签
- [ ] 已同时记录 `APP_VERSION`
- [ ] 已同时记录 `APP_COMMIT_SHA`
- [ ] 已同时记录 `APP_BUILD_TIME`
- [ ] 已确认私有仓库中镜像可拉取

## 3. 数据库与初始化

- [ ] 已确认本版本是否涉及数据库结构变化
- [ ] 如涉及结构变化，已补迁移说明
- [ ] 首次部署入口仍使用 `scripts/init-db/02-schema-consolidated.sql`
- [ ] 已确认默认角色、菜单、基础配置初始化无缺项

## 4. 配置与部署

- [ ] 已确认部署目录与源码目录分离
- [ ] 已确认 `.env` 或部署环境变量完整
- [ ] 已确认客户域名、CORS、对象存储代理前缀等环境配置正确
- [ ] 已确认备份目录或备份目标可用
- [ ] 已确认回滚所需上一稳定版本镜像仍可拉取

## 5. 测试与验收

- [ ] 已执行最小回归集
- [ ] 已执行发布后冒烟测试
- [ ] 已更新测试台账
- [ ] 已更新缺陷台账
- [ ] 已确认本次遗留问题和风险范围

最小回归集建议至少覆盖：

- 登录
- 入库
- 正式归档
- 检索
- 借阅申请
- 借阅审批
- 公开访问
- 备份恢复页面可用性

## 6. 交付留痕

- [ ] 已记录客户名称
- [ ] 已记录环境名称
- [ ] 已记录部署版本
- [ ] 已记录镜像标签
- [ ] 已记录提交号
- [ ] 已记录执行人和执行时间
- [ ] 已记录升级前备份编号
- [ ] 已记录是否回滚

## 7. 发布结论

- [ ] 允许发布
- [ ] 有条件发布
- [ ] 暂缓发布

如选择“有条件发布”或“暂缓发布”，应补充：

- 原因
- 风险范围
- 临时处置方案
- 下次复核时间
