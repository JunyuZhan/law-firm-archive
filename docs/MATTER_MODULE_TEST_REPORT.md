## 发现的Bug

### Bug #1: 结案审批时数据库事务错误

**严重程度**: 🔴 高

**错误信息**:
```
ERROR: current transaction is aborted, commands ignored until end of transaction block
SQL state [25P02]; error code [0]
```

**位置**: `approveCloseMatter` 方法

**原因分析**:
- 在结案审批通过后，系统尝试自动创建档案记录
- 创建档案过程中某个SQL操作失败，导致事务被标记为中断
- 后续的客户信息查询在同一个事务中执行，触发了"事务已中断"错误

**建议修复**:
1. 检查 `ArchiveAppService.createArchive` 方法的SQL操作
2. 考虑将档案创建放到独立事务中执行
3. 或者使用 `@Transactional(propagation = REQUIRES_NEW)` 隔离事务

## 已验证的业务规则

### 项目创建规则

1. **必须关联合同** - 项目只能基于已审批通过的合同创建
2. **客户一致性** - 项目客户必须与合同客户一致
3. **编号自动生成** - 格式：年份+创建人标识+类型码+序号
4. **状态初始化** - 基于已审批合同创建的项目直接进入"进行中"状态
5. **参与人自动添加** - 创建者自动添加为项目参与人

### 状态流转规则（严格状态机）

```
DRAFT (草稿) → PENDING (待审批)
              ↓
PENDING → ACTIVE (进行中) / DRAFT (退回)
              ↓
ACTIVE → SUSPENDED (暂停) / PENDING_CLOSE (待结案)
              ↓
SUSPENDED → ACTIVE (恢复) / PENDING_CLOSE (待结案)
              ↓
PENDING_CLOSE → CLOSED (已结案) / ACTIVE (驳回)
              ↓
CLOSED → ARCHIVED (已归档)
              ↓
ARCHIVED → 终态（不可变更）
```

### 删除与归档规则

1. **项目不可删除** - 只能通过状态变更为ARCHIVED归档
2. **已归档不可编辑** - 归档后的项目只读
3. **归档状态不可逆** - ARCHIVED是终态

### 团队管理规则

1. **自动添加创建者** - 创建项目时自动添加
2. **防止重复添加** - 同一用户不能重复添加
3. **只有参与者可编辑** - 非项目成员无法编辑项目
